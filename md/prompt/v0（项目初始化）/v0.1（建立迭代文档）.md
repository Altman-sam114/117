# v0.1（建立迭代文档）

## 版本号

v0.1

## 版本分配依据

人工未指定版本号。本轮目标是建立项目级 Agent 协作系统、记忆文档、测试规范、核心流程文档和提示词目录，属于项目初始化阶段，因此自动分配为 `v0.1`。

## 背景

MD Journal 已有一个可运行的 SwiftUI iOS Markdown 日记应用，包含本地 JSON 持久化、列表、编辑器、Markdown 预览、分类、心情、统计和横屏响应式布局。当前缺少稳定的多 Agent 迭代文档体系，后续容易出现上下文丢失、测试口径不一致、核心逻辑文档滞后等问题。

## 目标

- 创建标准入口 `AGENT.md`。
- 创建 `update_log.md`。
- 创建 `md/prompt/README.md` 和本轮提示词文件。
- 创建 `md/test/test.md`。
- 创建 `md/flow/flow.md`。
- 创建 `md/flow/flowchart.md`。
- 更新 README，使后续维护入口指向标准文档体系。
- 保留小写 `agent.md` 的兼容价值，但标准入口应为大写 `AGENT.md`。

## 非目标

- 不开发新的 App 功能。
- 不修改 Swift 源码行为。
- 不新增 XCTest target。
- 不引入 markdown lint 或第三方工具。
- 不删除用户数据或历史 git 记录。

## 架构依据

- `ContentView` 使用 `NavigationSplitView` 组织列表与详情。
- `JournalStore` 是 `@MainActor ObservableObject`，负责 `entries`、保存错误、本地 JSON 加载和保存。
- `JournalEntry` 是日记模型，并兼容旧 JSON 缺失字段。
- `EntryListView` 负责搜索、分类筛选、列表和统计入口。
- `EntryEditorView` 负责元数据编辑、正文编辑、Markdown 快捷插入和宽屏双栏。
- `MarkdownBlockParser` 负责轻量 Markdown 分块和 `###` 小节分组。
- `MarkdownPreviewView` 根据解析结果渲染预览。
- `JournalStatistics` 负责统计计算。

## 实现步骤

1. 阅读 README、小写 `agent.md`、git 记录和核心 Swift 文件。
2. 创建 `md/prompt`、`md/test`、`md/flow` 目录。
3. 写入 `AGENT.md`，保持入口规则精简。
4. 写入 `update_log.md`，记录当前状态、关键决策、历史维护记录和本轮 v0.1。
5. 写入 `md/test/test.md`，建立 Probe / Fast、Smoke、Stage Regression、Full 分层。
6. 写入 `md/flow/flow.md`，描述当前真实数据流、执行流、模块边界、测试映射和扩展点。
7. 写入 `md/flow/flowchart.md`，至少包含项目核心逻辑图和 Agent 迭代流程图。
8. 写入 `md/prompt/README.md` 和本文件。
9. 更新 README 和小写 `agent.md`，避免入口分叉。
10. 运行 `git diff --check`，并根据文档-only 范围运行项目静态验证。
