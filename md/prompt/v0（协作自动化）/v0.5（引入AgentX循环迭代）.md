# v0.5（引入 Agent X 循环迭代）

## 版本号

v0.5

## 版本分配依据

人工指定本轮版本号为 `v0.5`，版本标题为“引入 Agent X 循环迭代文档基线”。当前项目已有 Agent A/B/C 云端验证闭环，本轮只把文档体系改造成未来可由 Agent X 主控多轮调用 A/B/C 的准备状态，因此属于 `v0.x` 协作自动化阶段的小版本递增。

## 背景

项目当前流程是人工提出目标，Agent A 写提示词，Agent B 在 `main` 上实现并 push 到 `origin/main`，GitHub Actions 生成未加密结果包，Agent C 下载 artifact 并核对 manifest、JUnit 或等价摘要、日志和结果包产物。

未来需要新增 Agent X：Agent X 接收人工总目标 X，拆成多个有限轮次，并在每轮结束后根据 Agent C 的云端 artifact 验收结果判断继续、退回、暂停或完成。

## 本轮目标

- 在 `AGENTS.md` 增加 Agent X 的召唤规则、职责、循环边界、停止条件、禁止项和最终回复格式。
- 在 `md/flow/flow.md` 增加 Agent X 主控循环说明。
- 在 `md/flow/flowchart.md` 更新 Mermaid 图，表达人工给 Agent X 总目标、Agent X 拆分轮次、Agent A 写提示词、Agent B 实现并 push、GitHub Actions 生成 artifact、Agent C 验收 artifact、Agent X 判断继续/退回/暂停/完成。
- 在 `md/test/test.md` 增加 Agent X 循环下的验证规则和小数据量 artifact 下载限制。
- 在 `md/prompt/README.md` 增加 Agent X 相关提示词管理规则。
- 在 `README.md` 的协作说明中简短补充 `agentx:`。
- 在 `update_log.md` 增加 v0.5 记录。

## 后续 Agent X 使用方式

后续人工可用 `agentx:`、`x:` 或 `X:` 提供总目标。Agent X 接收后必须：

1. 拆分有限轮次目标。
2. 要求 Agent A 为每轮写版本化提示词。
3. 要求 Agent B 在 `main` 上实现、轻量检查、commit 并 push 到 `origin/main`。
4. 等待 GitHub Actions 生成最新未加密 artifact。
5. 要求 Agent C 下载并核对最新 artifact。
6. 根据 Agent C 结论决定继续下一轮、退回 Agent B 修复、暂停等待人工或宣布总目标完成。

## 风险和禁止项

- 禁止 Agent X 无条件无限循环。
- 禁止跳过 Agent C 云端 artifact 验收。
- 禁止把旧 run、旧 artifact、本地输出或文字汇报冒充最新云端结果。
- 禁止在总目标未完成时宣布完成。
- 禁止为了推进循环扩大无关改动范围。
