# v0.8（Markdown 预览解析单次化）

## 版本号

v0.8

## 版本分配依据

人工总目标要求“优美的写入，高效的性能”。`v0.7` 已把编辑写盘从每次输入立即保存优化为短延迟合并保存，并通过云端 artifact 验收。当前大正文编辑时，宽屏会同时显示编辑区和预览，`MarkdownPreviewView` 在同一次渲染中分别计算 `blocks`、`sectionGroups` 和 `shouldUseSectionGroups`，而 `groupedByLevelThree(_:)` 内部又会再次调用 `parse(_:)`。本轮继续沿 `v0.x` 性能优化小版本推进，先把预览解析收敛为一次。

## 背景

当前预览链路是：

```text
MarkdownPreviewView.body
  -> blocks: MarkdownBlockParser.parse(markdown)
  -> sectionGroups: MarkdownBlockParser.groupedByLevelThree(markdown)
  -> groupedByLevelThree 内部再次 parse(markdown)
```

在长正文、Mac 键盘高速输入和宽屏编辑/预览并排时，这会放大轻量 Markdown 解析成本。需要不改变 Markdown 语义、不引入第三方依赖的前提下，复用同一次解析结果。

## 本轮目标

- 新增一个轻量解析结果结构，用同一份 `[MarkdownBlock]` 派生 `###` 小节分组。
- `MarkdownPreviewView` 在单次 body 渲染中只调用一次 Markdown 文档解析。
- 保持现有块级 Markdown 支持不变：标题、段落、引用、列表、待办、代码块、分割线。
- 保持 `###` 小节分组语义不变，包括开篇内容、空三级标题回退为“未命名小节”。
- 新增或更新 XCTest，证明单次解析结果的 blocks 和 sectionGroups 与现有语义一致。
- 同步更新 README、测试规范、核心流程、流程图和更新日志中受影响部分。

## 非目标

- 不做完整 Markdown 引擎，不引入 CommonMark 或第三方依赖。
- 不改变 `JournalEntry` JSON 格式、持久化路径或 `JournalStore` 写入策略。
- 不缓存列表卡片 `entry.sections`、统计计算或跨 entry 的派生数据；这些可作为后续轮次。
- 不改变预览 UI 视觉结构。
- 不新增 native macOS target；Mac 路径仍是 Mac Catalyst。

## 架构依据

- `MarkdownBlockParser` 是轻量 Markdown 解析边界，解析复用应放在该层或其直接结果类型中。
- `MarkdownPreviewView` 只消费解析结果并渲染，不负责保存数据。
- `EntryEditorView` 继续只向预览传入正文和 accent，不持有解析缓存状态。
- XCTest 应覆盖规则层，不依赖 SwiftUI 运行或模拟器交互。

## 实现步骤

1. 在 `MarkdownBlockParser.swift` 中新增解析结果类型，例如 `MarkdownParseResult`，包含：
   - `blocks: [MarkdownBlock]`
   - `sectionGroups: [MarkdownSectionGroup]`
   - `shouldUseSectionGroups: Bool`
2. 新增 `MarkdownBlockParser.parseDocument(_:)`：
   - 只调用一次 `parse(_:)`。
   - 用解析后的 blocks 派生 `sectionGroups`。
   - 保留 `groupedByLevelThree(_:)` 作为兼容 API，但内部复用同一个 blocks 派生函数。
3. 调整 `MarkdownPreviewView`：
   - 在 `body` 中用 `let document = MarkdownBlockParser.parseDocument(markdown)`。
   - 后续渲染只使用 `document.blocks`、`document.sectionGroups` 和 `document.shouldUseSectionGroups`。
4. 更新 `MarkdownBlockParserTests`：
   - 验证 `parseDocument(_:)` 的 blocks 与 `parse(_:)` 一致。
   - 验证 `parseDocument(_:)` 的 sectionGroups 保持 `###` 分组语义。
5. 将 CI 结果包版本更新为 `v0.8`，方便 Agent C 核对本轮云端结果。
6. 同步 README、flow、flowchart、test 和 update log。

```sh
git diff --check
git diff --cached --check
plutil -lint MDJournal.xcodeproj/project.pbxproj
xcrun swiftc -parse -parse-as-library $(rg --files -g '*.swift' MDJournal)
ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
```

由于修改 `MarkdownBlockParserTests`，尝试本机 XCTest；若 CoreSimulatorService 仍不可用，记录关键错误并以 GitHub Actions artifact 为最终重验证依据。可补 generic iOS 和 Mac Catalyst build，确认 Mac 版本路径不回归。

Agent C 必须下载最新 run 的 artifact，并核对：

- `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 全部为 `success`。

- 禁止把解析逻辑塞进 SwiftUI 视图层。
- 禁止引入第三方依赖。
