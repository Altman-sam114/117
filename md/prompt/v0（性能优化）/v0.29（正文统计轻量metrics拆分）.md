# v0.29（正文统计轻量 metrics 拆分）

## 版本分配依据

- 当前最新正式记录为 `v0.28`。
- 本轮继续服务 Agent X 总目标中的“高效性能”和 Mac Catalyst 长文写作体验，属于 `v0.x` 性能优化小步迭代，因此递增为 `v0.29`。

## 背景

`JournalEntryBodySummary` 当前同时计算摘要、词数、`###` 小节和小节数。列表行和编辑器头部需要摘要，但 `JournalStatistics` 与 `JournalListOverviewSnapshot` 只需要词数和小节信息。统计和列表概览路径继续构造 summary 会额外生成 excerpt，对长正文编辑后的刷新不够轻。

## 目标

- 新增非持久化 `JournalEntryBodyMetrics`，只计算 `wordCount`、`sections` 和 `sectionCount`。
- `JournalEntry` 新增 `bodyMetrics` 计算属性。
- `JournalEntryBodySummary` 继续负责 `excerpt + metrics`，并复用 `JournalEntryBodyMetrics`，避免词数和 `###` 小节规则重复实现。
- `JournalStatistics` 和 `JournalListOverviewSnapshot` 改用 `entry.bodyMetrics`，不再为统计/概览生成 excerpt。
- 扩展 `JournalEntryTests`，验证 `bodyMetrics` 与 `bodySummary.metrics` 的指标一致。
- `.github/workflows/ci-results.yml` 的 `VERSION` 升为 `v0.29`。
- 同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 JSON schema、`Codable` 字段、本地文件名或 `JournalStore` 持久化链路。
- 不改变摘要生成规则、词数 split 规则、`### ` 小节识别规则、统计口径、概览 insight 文案或 UI 样式。
- 不新增第三方依赖，不修改 deployment target。
- 不触碰 Xcode project，不新增独立 Swift 文件。
- 不运行本地完整 build、XCTest、simulator 或 app。

## 架构依据

- `JournalEntry` 仍是日记模型和兼容解码边界，metrics 与 summary 都是非持久化派生数据。
- `JournalSection.extract(from:)` 仍是唯一 `###` 小节提取入口。
- `JournalStatistics` 只做统计聚合，视图只消费统计结果。
- `JournalListOverviewSnapshot` 继续作为列表首页轻量概览，不构造完整统计看板数据。

## 实现步骤

1. Agent B 开始前执行 `git fetch origin`、切到 `main`、`git pull --ff-only origin main`，确认工作区无无关改动。
2. 在 `JournalEntry.swift` 中新增 `JournalEntryBodyMetrics: Equatable`：
   - `let wordCount: Int`
   - `let sections: [JournalSection]`
   - `var sectionCount: Int { sections.count }`
   - `init(body:)` 使用现有词数 split 和 `JournalSection.extract(from:)`。
3. 给 `JournalEntry` 新增 `var bodyMetrics: JournalEntryBodyMetrics { JournalEntryBodyMetrics(body: body) }`。
4. 保留 `bodySummary`、`excerpt`、`wordCount`、`sections`、`sectionCount` 旧 API。
5. 调整 `JournalEntryBodySummary`，持有 `metrics`，并让 `wordCount`、`sections`、`sectionCount` 委托给 metrics。
6. `JournalStatistics.init` 中使用 `entry.bodyMetrics`，所有统计字段从 metrics 取值。
7. `JournalListOverviewSnapshot.init` 中使用 `entry.bodyMetrics`，概览路径避免生成 excerpt。
8. `EntryRowView`、`EntryEditorView` 继续使用 `entry.bodySummary`，因为它们仍需要摘要或展示用派生数据。
9. 扩展 `JournalEntryTests`，断言同一 body 下 `bodyMetrics` 与 `bodySummary.metrics` 的词数、小节和小节数一致。
10. 更新 workflow 版本和相关文档。
