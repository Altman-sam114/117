# v0.51（Markdown 解析空行判断非分配）

## 版本号

`v0.51`

## 版本分配依据

- 当前最新已验收版本为 `v0.50`，本轮继续围绕 Agent X 总目标中的长文写作、Mac Catalyst 宽屏实时预览和性能优化推进。
- 本轮只优化 `MarkdownBlockParser` 的空行判断内部实现，不改变 Markdown 解析外部语义，因此沿 `v0.x` 小版本递增为 `v0.51`。

## 背景

`MarkdownPreviewView` 在编辑正文时会调用 `MarkdownBlockParser.parseDocument(_:)` 解析正文。当前 `MarkdownBlockParser.parse(_:)` 对每一行执行：

```swift
rawLine.trimmingCharacters(in: .whitespaces).isEmpty
```

用于判断空白行。长正文预览中这个判断会随每一行重复创建临时 trimmed 字符串。本轮将其替换为直接扫描 unicode scalar 的水平空白判断，减少空行识别的中间分配。

## 目标

- 将 `MarkdownBlockParser.parse(_:)` 的空行判断改为非分配扫描。
- 保持原语义：空字符串、普通空格、tab 和 `CharacterSet.whitespaces` 覆盖的水平空白行视为空白行。
- 不扩大到按换行字符重新切分；现有 `components(separatedBy: .newlines)` 分行语义保持不变。
- 扩展 `MarkdownBlockParserTests` 覆盖水平空白行会分隔段落、代码块内空白行会保留。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不重写 `MarkdownBlockParser` 的整套分行策略。
- 不改变标题、引用、无序列表、有序列表、待办、代码块、分割线或 `###` 小节分组规则。
- 不改变 `MarkdownPreviewView` 渲染、内联 Markdown 快路径或索引迭代。
- 不改变 `JournalEntry`、`JournalStore`、本地 JSON 持久化、统计口径或 Mac Catalyst 菜单行为。
- 不引入第三方依赖，不修改 deployment target。
- 不运行本机完整 build、XCTest、模拟器或 app；按人工要求最终只用 GitHub Actions artifact 重验证。

## 架构依据

- `MarkdownBlockParser` 是轻量 Markdown 块解析边界，预览层只消费解析结果。
- 空行判断属于解析器内部实现细节，适合在该层优化，不应上移到 SwiftUI 视图或持久化层。
- 本轮只减少每行空白判断的临时字符串，不改变块级 Markdown 契约。

- `MDJournalTests/MarkdownBlockParserTests.swift`
  - 增加水平空白行分隔段落测试。
  - 增加代码块内水平空白行保留测试。

  - 更新 `VERSION` 为 `v0.51`。
- `README.md`
- `md/test/test.md`

## 实现步骤

1. 确认当前分支为 `main` 且同步最新 `origin/main`。
2. 在 `MarkdownBlockParser.swift` 中新增私有 helper：
   - 对 `unicodeScalars` 执行 `allSatisfy { CharacterSet.whitespaces.contains($0) }`。
   - 空字符串自然返回 `true`，保持原空行语义。
3. 将 `parse(_:)` 中空行判断替换为 helper。
4. 增加测试：
   - 段落之间的空格、tab 或其它水平空白行应 flush 段落。
   - fenced code block 内的水平空白行应作为 code 内容保留，不触发普通块 flush。
5. 更新 workflow 版本和文档。
