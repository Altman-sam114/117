# v0.38（编辑器占位非分配判断）Agent A 提示词

## 版本号和分配依据

- 版本号：`v0.38`。
- 分配依据：当前最新已验收版本为 `v0.37`，总目标仍是 Mac Catalyst 可用、界面优雅和长文写入高效。本轮继续沿 `v0.x` 做一个小范围 SwiftUI 写入性能优化。

## 背景

`EntryEditorView` 的正文编辑区域为了显示 placeholder，会在 SwiftUI `body` 渲染时用 `entry.body.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty` 判断正文是否为空。这个写法会为正文创建临时字符串；长文输入时编辑器重渲染频繁，placeholder 判断不应该引入额外分配。

## 目标

- 将正文 placeholder 的空白判断改为非分配字符扫描。
- 保持原语义：空字符串或全空白/换行正文仍显示“用 ### 小节组织今天的记录。”；只要有非空白字符就隐藏 placeholder。
- 将 GitHub Actions 结果包版本更新为 `v0.38`。
- 同步 README、测试规范、核心流程、流程图、更新日志和本提示词。

## 非目标

- 不修改 `MarkdownBodyTextView`、正文 binding、光标/选区同步或键盘处理。
- 不修改 `JournalEntry` 的持久化字段、JSON 解码或保存路径。
- 不修改 Markdown 解析、`###` 小节识别、预览、统计或列表搜索语义。
- 不新增第三方依赖，不新增独立 native macOS target。
- 不运行本机完整 build、XCTest、模拟器或 GUI app；人工已要求通过云端回传结果包验证。

## 架构依据

- `EntryEditorView` 负责正文编辑布局和 placeholder 展示。
- `JournalStore` 仍是唯一日记集合修改与本地持久化入口，本轮不触碰。
- placeholder 判断是纯展示派生，适合收敛在 `EntryEditorView` 文件内，避免扩大模型或持久化边界。

## 实现步骤

1. 在最新 `origin/main` 的 `main` 分支开始。
2. 在 `EntryEditorView.swift` 中把 placeholder 条件从 `trimmingCharacters(...).isEmpty` 改为非分配 helper，例如 `entry.body.contains { !$0.isWhitespace }`。
3. helper 保持 fileprivate/private 范围，不改变 `JournalEntry` Codable 或公共持久化结构。
4. 将 workflow `VERSION` 更新为 `v0.38`。
5. 同步文档，说明编辑器正文 placeholder 判断不再创建临时字符串。
