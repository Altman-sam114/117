# v0.49（Markdown 回车空项退出非分配判断）

## 版本号

`v0.49`

## 版本分配依据

- 当前最新提示词和更新日志记录到 `v0.48`，本轮继续围绕 Agent X 性能优化目标做小步改进，因此递增为 `v0.49`。
- 本轮只优化 `MarkdownLineContinuation` 中空列表/引用/待办/有序列表项的退出判断，属于纯规则层内部实现优化，不改变用户可见 Markdown 输入语义，不开新大版本。

## 背景

`MarkdownLineContinuation` 在用户按回车时负责无序列表、待办、引用和有序列表的续写或退出。当前 `isEmptyContinuationLine(prefix:lineSuffix:)` 会把 `prefix.contentBeforeCursor` 和 `lineSuffix` 从 `Substring` 转为 `String`，再调用 `trimmingCharacters(in: .whitespaces).isEmpty` 判断当前项是否为空。

这会在长文写作、列表中间拆分、空项退出等高频输入路径上产生临时字符串。本轮要求保留现有语义，只把判断方式改为直接扫描 `Substring`，避免 `String` 转换和 trim 产生的额外分配。

## 目标

- 优化 `MarkdownLineContinuation.isEmptyContinuationLine(prefix:lineSuffix:)`。
- 将 `String(substring).trimmingCharacters(in: .whitespaces).isEmpty` 改为对 `Substring` 的非分配 whitespace 扫描。
- 保持现有语义：只把 `CharacterSet.whitespaces` 对应的水平空白视为空内容。
- 明确换行不应出现在 `prefix.contentBeforeCursor` 或 `lineSuffix` 片段中；本轮不扩大为空白换行判断。
- 保持无序列表、待办、引用和有序列表的回车续写、空项退出、行中拆分、UTF-16 光标和 fenced code 行为不变。
- 补充或调整 `MarkdownLineContinuationTests`，覆盖空项退出判断的空格、tab、非空内容和行中后缀边界，确保语义等价。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志中受影响的当前状态描述。

## 非目标

- 不改变 `continuationPrefix(in:)` 对无序列表、待办、引用或有序列表前缀的识别规则。
- 不改变有序列表编号递增、`Int.max` 溢出保护或缩进保留规则。
- 不改变 `isInsideFencedCodeBlock(before:in:)` 和 `lineStartsFence(_:)` 的单次扫描实现。
- 不改变 Tab / Shift-Tab 缩进、Markdown 片段插入、Markdown 预览、统计、列表筛选、JSON 持久化或保存节流。
- 不把 `.whitespaces` 改成 `.whitespacesAndNewlines`，也不使用 `Character.isWhitespace` 扩大语义。
- 不引入第三方依赖，不修改 deployment target，不做无关重构。

## 架构依据

- `MarkdownLineContinuation` 是正文回车续写的纯规则层，消费 `String`、UTF-16 `NSRange` 和 replacement text，输出可选的新正文与选区。
- 空项退出判断属于该规则层内部细节，应在 `MarkdownLineContinuation` 内部优化，不下沉到 `MarkdownBodyTextView`，也不触碰 SwiftUI/UIKit bridge。
- `JournalStore` 是唯一持久化入口；本轮不涉及日记集合修改、本地 JSON 格式或保存链路。
- Markdown 预览由 `MarkdownBlockParser` 和 `MarkdownPreviewView` 负责；本轮不改变预览解析或渲染。

## 影响范围和关键文件

- `MDJournal/Utilities/MarkdownLineContinuation.swift`
  - 替换 `isEmptyContinuationLine(prefix:lineSuffix:)` 内部的 `String` + `trimmingCharacters(in:)` 判断。
  - 新增私有 helper，例如 `isWhitespaceOnly(_ text: Substring) -> Bool`，直接遍历 `Substring` 中的字符。
  - helper 必须使用等价于 `CharacterSet.whitespaces` 的判断，不把换行视为空，也不只缩窄为普通空格和 tab。
- `MDJournalTests/MarkdownLineContinuationTests.swift`
  - 补充空项退出语义测试：仅空格、tab 或二者混合时退出；存在非空正文或当前行光标后仍有正文时不退出而应续写/拆分。
  - 保留既有 fenced code、非折叠选区、普通输入、emoji/UTF-16 光标等测试行为。
- `.github/workflows/ci-results.yml`
  - 更新 `VERSION` 为 `v0.49`，确保 manifest 与 artifact 名称对应本轮 commit。

- `md/test/test.md`

- `md/flow/flow.md`

- `md/flow/flowchart.md`

## 实现步骤

1. 开始前按项目规则读取入口文档、测试规范、本提示词和相关源码，并同步最新 `origin/main`。
2. 在 `MarkdownLineContinuation.swift` 中定位 `isEmptyContinuationLine(prefix:lineSuffix:)`。
3. 新增私有非分配 helper，直接扫描 `Substring`：
   - 空 `Substring` 返回 `true`。
   - 每个 unicode scalar 都必须属于 `CharacterSet.whitespaces` 才返回 `true`。
   - 不使用 `String` 转换、`trimmingCharacters(in:)`、`split` 或其它会为判断创建临时字符串/片段集合的写法。
   - 在注释或测试中锁定“不包含换行”的前提；如果将来传入换行，也不应被当成本轮退出判断里的空内容。
4. 用该 helper 替换 `isEmptyContinuationLine` 的两处 `String(...).trimmingCharacters(in: .whitespaces).isEmpty`。
5. 不改变 `lineStart(before:in:)`、`lineEnd(after:in:)`、`isInsideFencedCodeBlock(before:in:)`、`continuationPrefix(in:)` 和 `removingContinuationPrefix` 的行为。
6. 扩展 `MarkdownLineContinuationTests`：
   - 空 bullet / checklist / ordered list / quote 现有退出用例必须继续通过。
   - 增加包含 tab 或空格 + tab 的空项退出用例。
   - 增加光标后 `lineSuffix` 只有空格或 tab 时仍退出的用例。
   - 增加光标前或光标后存在非空字符时不退出、而是续写并拆分当前行的用例。
7. 更新 `.github/workflows/ci-results.yml` 的 `VERSION` 为 `v0.49`。
8. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md` 中受影响描述，避免文档落后于真实行为。
9. 只提交本轮相关文件，不扩大范围。

- 禁止把 `.whitespaces` 语义扩大为 `.whitespacesAndNewlines` 或 `Character.isWhitespace`。

- 禁止引入第三方依赖、修改 deployment target 或扩大到 Markdown 预览/统计/列表重构。
