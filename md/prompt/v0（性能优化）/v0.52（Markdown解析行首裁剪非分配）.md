# v0.52（Markdown 解析行首裁剪非分配）

## 版本号

`v0.52`

## 版本分配依据

- 当前最新已验收版本为 `v0.51`，本轮继续围绕 Agent X 总目标中的长文写作、Mac Catalyst 宽屏实时预览和 Markdown 解析性能推进。
- 本轮只优化 `MarkdownBlockParser` 行首空格/tab 裁剪的内部实现，不改变分行策略或块级 Markdown 外部语义，因此沿 `v0.x` 小版本递增为 `v0.52`。

## 背景

`MarkdownBlockParser.parse(_:)` 当前在遍历每一行时会调用 `trimmingLeadingWhitespace()`，该 helper 通过 `String(drop(...))` 为每行构造一个新的 `String`，再用于判断 heading、quote、列表、待办、代码围栏和分割线。长正文实时预览时，这会随每一行重复分配临时字符串。

v0.51 已把空行判断改为非分配扫描。本轮进一步把行首裁剪结果改为 `Substring`，让 marker 判断直接在原行切片上完成，只在最终写入 block 内容时转换为 `String`。

## 目标

- 将 `MarkdownBlockParser` 的行首空白裁剪 helper 改为返回 `Substring`。
- 将 heading、quote、无序列表、有序列表、待办解析 helper 改为接收 `Substring`。
- 保持原有语义：只跳过普通空格和 tab，不扩大到所有 Unicode whitespace。
- 保持 `components(separatedBy: .newlines)` 分行策略不变，避免本轮触碰 CR/CRLF 归一化风险。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不重写整套 Markdown 分行器。
- 不改变 CR/CRLF、CR-only、尾随换行或段落 join 语义。
- 不改变标题、引用、无序列表、有序列表、待办、代码块、分割线或 `###` 小节分组规则。
- 不改变 `MarkdownPreviewView` 渲染、内联 Markdown 快路径或索引迭代。
- 不改变 `JournalEntry`、`JournalStore`、本地 JSON 持久化、统计口径或 Mac Catalyst 菜单行为。
- 不引入第三方依赖，不修改 deployment target。
- 不运行本机完整 build、XCTest、模拟器或 app；按人工要求最终只用 GitHub Actions artifact 重验证。

## 架构依据

- `MarkdownBlockParser` 是轻量 Markdown 块解析边界，预览层只消费解析结果。
- 行首裁剪属于解析器内部 marker 识别细节，适合在该层优化。
- 只把临时 `String` 改为 `Substring`，不会改变 `MarkdownBlock` 输出类型或视图渲染接口。

## 影响范围和关键文件

- `MDJournal/Utilities/MarkdownBlockParser.swift`
  - 将私有 `trimmingLeadingWhitespace()` 替换为 `leadingWhitespaceTrimmed() -> Substring`。
  - 将 marker parser helper 参数从 `String` 改为 `Substring`。
- `MDJournalTests/MarkdownBlockParserTests.swift`
  - 复用既有缩进 heading / ordered list / code block 测试覆盖行为不变。
- `.github/workflows/ci-results.yml`
  - 更新 `VERSION` 为 `v0.52`。
- `README.md`
- `md/test/test.md`
- `md/flow/flow.md`
- `md/flow/flowchart.md`
- `update_log.md`

## 实现步骤

1. 确认当前分支为 `main` 且同步最新 `origin/main`。
2. 在 `MarkdownBlockParser.swift` 中把行首裁剪 helper 改为返回 `Substring`。

4. 保持最终 block 数据仍为 `String` / `[String]` / `OrderedListItem` / `ChecklistItem`。
