# v0.37（预览索引迭代）Agent A 提示词

## 版本号和分配依据

- 版本号：`v0.37`。
- 分配依据：当前最新已验收版本为 `v0.36`，总目标仍是 Mac Catalyst 可用、界面优雅和长文写入高效。本轮继续沿 `v0.x` 做一个小范围 SwiftUI 性能优化。

## 背景

`MarkdownPreviewView` 已在 `v0.8` 改为单次调用 `MarkdownBlockParser.parseDocument(_:)`，并由 `MarkdownParseResult` 渲染普通块和 `###` 小节分组。当前预览中多处 `ForEach(Array(...enumerated()), id: \.offset)` 会在 SwiftUI `body` 反复计算时为块序列和列表项创建临时数组。对长正文、宽屏实时预览和 Mac Catalyst 写作场景，这属于可以避免的额外分配。

## 目标

- 将 `MarkdownPreviewView` 中普通块、小节内块、无序列表、有序列表和待办列表的 `ForEach(Array(...enumerated()), id: \.offset)` 改为基于 `indices` 的 `ForEach`。
- 保持原本 offset 身份语义：列表项和块仍按当前位置作为 SwiftUI id。
- 保持 UI、文案、颜色、间距、空内容展示、有序列表编号显示和 `###` 小节分组行为不变。
- 将 GitHub Actions 结果包版本更新为 `v0.37`。
- 同步 README、测试规范、核心流程、流程图、更新日志和本提示词。

## 非目标

- 不修改 `MarkdownBlockParser` 的解析规则。
- 不修改 `JournalEntry`、`JournalStore`、本地 JSON 格式或持久化路径。
- 不新增第三方依赖。
- 不新增独立 native macOS target；Mac 版本仍走 Mac Catalyst。
- 不运行本机完整 build、XCTest、模拟器或 GUI app；人工已要求通过云端回传结果包验证。

## 架构依据

- `MarkdownPreviewView` 只消费解析结果并渲染预览，不负责保存数据。
- `MarkdownBlockParser` 仍是 Markdown 块和 `###` 小节分组的唯一解析边界。
- 本轮只改变 SwiftUI `ForEach` 数据源形式，不改变 `MarkdownBlock`、`MarkdownSectionGroup` 或用户数据。

## 实现步骤

1. 在最新 `origin/main` 的 `main` 分支开始。
2. 替换 `MarkdownPreviewView` 中所有 `ForEach(Array(...enumerated()), id: \.offset)`：
   - `document.blocks.indices`
   - `group.blocks.indices`
   - `items.indices`
3. 列表项需要多次读取时，在 `ForEach` closure 内用 `let item = items[index]`，不改变展示内容。
4. 将 workflow `VERSION` 更新为 `v0.37`。
5. 同步文档，说明预览渲染改为索引迭代以减少临时数组分配。
