# v0.36 / 小节概览懒加载

## 版本分配依据

- 当前最新正式版本为 `v0.35`。
- 本轮继续推进总目标“做个 Mac 版、优美写入、高效性能”，沿 `v0.x` 小版本递增为 `v0.36`。
- 本轮属于正文编辑热路径 SwiftUI 渲染性能优化，放入 `md/prompt/v0（性能优化）/`。

## 背景

`EntryEditorView` 头部已经在 `v0.34` 改为消费轻量 `JournalEntryBodyMetrics`，避免为了头部统计生成未展示的正文摘要。但 `JournalSectionOverview` 在展示多个 `###` 小节时仍使用横向 `ScrollView + HStack + ForEach(sections)`，会 eager 构建全部小节卡片。

每张小节卡片都会读取 `section.excerpt`。当用户写长文并拆出较多 `###` 小节时，正文变化会触发编辑器刷新，离屏小节卡片也会被同步构建。把横向栈改为 `LazyHStack` 能让离屏小节卡片按需构建，降低正文输入热路径上的渲染和字符串派生成本。

## 目标

- 将 `JournalSectionOverview` 中横向小节列表的 `HStack` 改为 `LazyHStack`。
- 保持现有 `ForEach(sections)`、卡片样式、标题、excerpt、宽度和滚动行为不变。
- 更新 README、核心流程、流程图、测试规范、更新日志和 CI artifact 版本号。

## 非目标

- 不修改 `JournalSection.extract(from:)`、`section.excerpt`、`JournalEntryBodyMetrics` 或 `JournalEntryBodySummary` 语义。
- 不修改 Markdown 解析、预览、写作菜单、片段插入、行缩进或回车续写规则。
- 不修改 `JournalStore`、JSON 持久化、模型字段或兼容解码。
- 不新增第三方依赖，不新增 native macOS target。
- 不运行本机完整构建、XCTest、模拟器或 GUI；最终以 GitHub Actions artifact 为准。

## 架构依据

- `EntryEditorView` 负责编辑器头部和正文布局，是本轮唯一 SwiftUI 行为改动点。
- `JournalSectionOverview` 只消费 `JournalEntryBodyMetrics.sections` 并展示，不负责派生或保存数据。
- SwiftUI 性能规则建议大型或可能增长的 `ScrollView` 子内容使用 `LazyVStack` / `LazyHStack`，避免离屏子视图 eager 构建。
