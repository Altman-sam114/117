# v0.46（正文词数统计单次扫描）

## 版本分配依据

- 当前最新已验收版本为 `v0.45`，本轮继续围绕 Agent X 总目标中的长文写作和派生计算性能做小步优化，因此递增为 `v0.46`。
- 本轮只优化非持久化正文词数统计的中间分配，不改变用户可见语义，不开新大版本。

## 背景

`JournalEntryBodyMetrics` 是编辑器头部、列表概览、统计看板和列表卡片复用的轻量正文派生入口。当前词数统计使用 `body.split { $0.isWhitespace || $0.isNewline }.count`，可以得到正确结果，但会为长正文创建临时 `Substring` 片段集合。本轮将词数统计改为单次字符扫描，以保持原空白分隔语义并减少分配。

## 目标

- 将 `JournalEntryBodyMetrics` 的词数统计从 `split` 改为单次字符扫描。
- 保持原有空白分隔语义：连续空白、首尾空白、换行和 tab 不产生空词。
- 保持中文、英文和 Markdown 标记文本仍按空白边界计数。
- 扩展 `JournalEntryTests` 覆盖词数边界。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 `JournalSection.extract(from:)` 或 `###` 小节识别。
- 不改变正文摘要清理、Markdown 预览、列表筛选、统计聚合或 JSON 持久化格式。
- 不新增缓存状态，不把派生结果写入 `JournalEntry`。
- 不新增第三方依赖，不调整 deployment target。
- 不本机运行 app、模拟器、完整 build 或 XCTest；按人工要求最终只用 GitHub Actions artifact 重验证。

## 架构依据

- `JournalEntryBodyMetrics` 是非持久化正文 metrics 的集中入口。
- 编辑器头部、统计和列表概览在不需要摘要时优先使用 metrics，因此优化词数统计能覆盖多个长文高频派生路径。
- 词数统计属于模型派生层，不应进入 SwiftUI 视图、`JournalStore` 或持久化层。

## 影响范围和关键文件

- `MDJournal/Models/JournalEntry.swift`：新增 `JournalEntryBodyMetrics.wordCount(in:)`，并在 initializer 中使用单次扫描。
- `MDJournalTests/JournalEntryTests.swift`：补充词数边界测试。
- `.github/workflows/ci-results.yml`：更新 `VERSION` 为 `v0.46`。
- `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`：同步当前真实行为。
- `md/prompt/v0（性能优化）/v0.46（正文词数统计单次扫描）.md`：保存本轮 Agent A 提示词。

## 实现步骤

1. 在 `JournalEntryBodyMetrics` 中新增静态 `wordCount(in:)`。
2. 用 `isInsideWord` 布尔值扫描 `body` 字符：遇到空白重置，遇到非空白且当前不在词内时计数加一。
3. `init(body:)` 改为调用 `Self.wordCount(in:)`。
4. 增加测试覆盖空正文、全空白、连续空白、换行、tab、中文英文混排和 Markdown 标记文本。
5. 更新 workflow 版本和文档。
