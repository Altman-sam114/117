# v0.13（列表派生快照）

## 版本分配依据

当前最新正式版本为 `v0.12`，已完成 Mac Catalyst Markdown 片段菜单命令。人工总目标要求持续优化 Mac 版写作体验和高效性能；本轮沿 `v0.x` 小版本递增为 `v0.13`，聚焦列表搜索、分类筛选和分类计数的单次派生。

## 背景

`EntryListView` 当前在 `body`、标题、分类 chip 和 `ForEach` 中多次读取 `filteredEntries`，分类计数也会对 `entries` 重复过滤。随着日记数量增多，列表渲染、搜索和切换分类会重复遍历同一批日记。项目已有 `JournalEntryBodySummary` 和 `JournalStatistics` 的单次派生模式，本轮把列表筛选也收敛为同类非持久化快照。

## 目标

- 新增 `JournalEntryListSnapshot`，一次遍历生成：
  - `filteredEntries`
  - 全部日记数量
  - 每个分类的数量
  - 当前 section 标题
- `EntryListView` 在 `body` 中构造一次快照，并传给列表、section 标题和分类筛选 chip。
- 保持搜索语义不变：搜索文本先 trim，非空时匹配展示标题、正文、分类 rawValue 和心情 rawValue；分类筛选仍优先限制候选集。
- 保持分类 chip 数量语义不变：显示全部 entries 的分类分布，不受搜索文本影响。
- 补充 XCTest 覆盖搜索、分类筛选、分类计数和空白搜索。
- GitHub Actions artifact 版本更新为 `v0.13`。

## 非目标

- 不改变列表视觉布局、颜色、文案或排序。
- 不改变 `JournalEntry` JSON schema。
- 不改变 `JournalStore` 创建、删除、更新或保存路径。
- 不改变 Markdown 解析、统计口径或 `###` 小节规则。
- 不引入缓存状态、第三方依赖或后台索引。

## 架构依据

- `EntryListView` 继续负责列表、搜索、分类筛选和统计入口。
- `JournalEntryListSnapshot` 是非持久化派生数据，只消费 `[JournalEntry]`、搜索文本和选中分类，不持有状态，不写入 JSON。
- `JournalStore` 仍是唯一集合修改和本地持久化入口。
- 快照与 `JournalEntryBodySummary`、`JournalStatistics` 保持同样的“源数据 -> 单次派生 -> 视图消费”模式。

## 实现步骤

1. 新增 `JournalEntryListSnapshot`，在初始化时 trim 搜索文本，并单次遍历 entries 生成分类计数和过滤结果。
2. 提供 `count(for:)` 和 `sectionTitle` 等视图友好的只读 API。
3. 修改 `EntryListView.body`：构造 `let listSnapshot = JournalEntryListSnapshot(...)`，用它替代 `filteredEntries`、`sectionTitle` 和 `count(for:)`。
4. 保留 `EntryListView` 的 `@State searchText` 和 `selectedCategory`，不引入额外缓存状态。
5. 新增 `JournalEntryListSnapshotTests`，覆盖：
   - 空白搜索等同无搜索。
   - 搜索匹配标题、正文、分类和心情。
   - 分类筛选先限制结果。
   - 分类计数不受搜索影响。
   - section 标题使用过滤后的结果数量。
6. 更新 CI `VERSION` 为 `v0.13`。
7. 同步 README、核心流程、流程图、测试规范和更新日志。
