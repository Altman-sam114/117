# v0.10（正文派生与统计单次化）

## 版本号

v0.10

## 版本分配依据

人工总目标要求“优美的写入，高效的性能”。`v0.7` 已优化编辑写盘，`v0.8` 已优化 Markdown 预览解析，`v0.9` 已补齐 Mac Catalyst 菜单入口。当前剩余性能热点集中在列表、编辑器头部和统计看板对同一正文的重复派生计算。本轮继续沿 `v0.x` 性能优化推进。

## 背景

`JournalEntry.excerpt`、`wordCount`、`sections` 和 `sectionCount` 都从 `body` 派生。当前 `EntryRowView`、`EntryEditorView` 和 `JournalStatistics` 会在同一次渲染或统计初始化中多次访问这些 computed property，导致重复扫描正文和重复抽取 `###` 小节。统计看板和列表概览还会多次构造 `JournalStatistics`。

## 本轮目标

- 新增非持久化正文派生结果，例如 `JournalEntryBodySummary`，一次性计算摘要、词数、`###` 小节和小节数。
- `EntryRowView` 在单次 body 渲染中只构造一次正文派生结果，并复用摘要、词数和小节条。
- `EntryEditorView` 头部只构造一次正文派生结果，并复用词数、小节数和小节概览。
- `JournalStatistics` 在初始化时对每篇日记只构造一次正文派生结果，并用单轮聚合生成总量、分类、心情、本周和 7 天趋势。
- `EntryListView` 和 `StatisticsDashboardView` 在单次 body 中只构造一次 `JournalStatistics` 并向子视图传递。
- 保持现有 JSON、Markdown 语义、统计口径和界面视觉不变。
- 同步 README、测试规范、核心流程、流程图和更新日志中受影响部分。

## 非目标

- 不新增持久化字段，不缓存到 JSON，不做迁移。
- 不改变 `JournalSection.extract` 和 `MarkdownBlockParser` 的语义。
- 不重写搜索索引，不做跨 entry 的长期缓存。
- 不改变统计 UI 视觉结构。
- 不引入第三方依赖。

## 架构依据

- `JournalEntry` 是日记模型和派生展示信息边界，非持久化 summary 可以放在模型文件内，但不能进入 `CodingKeys`。
- `JournalStatistics` 是统计计算边界，视图只展示结果，不复制聚合逻辑。
- `EntryRowView` 和 `EntryEditorView` 只消费派生结果，不负责持久化。
- `JournalStore` 仍是唯一集合修改和本地保存入口，本轮不改写入策略。

## 实现步骤

1. 在 `JournalEntry.swift` 中新增 `JournalEntryBodySummary`，提供 `excerpt`、`wordCount`、`sections` 和 `sectionCount`。
2. 保留现有 `JournalEntry.excerpt`、`wordCount`、`sections`、`sectionCount` API，但让它们委托给 `bodySummary`，兼容现有调用。
3. 调整 `EntryRowView`，在 `body` 中构造一次 summary，并传给小节条函数。
4. 调整 `EntryEditorView`，在 `body` 中构造一次 summary，并传给 header、stat pills 和小节概览。
5. 重写 `JournalStatistics.init` 的内部聚合：每篇 entry 只计算一次 summary，用字典或聚合结构保存分类、心情、日期和本周统计。
6. 调整 `EntryListView` 和 `StatisticsDashboardView`，在单次 body 中构造一次 `JournalStatistics`，并传给子视图函数。
7. 更新或新增 XCTest，验证 summary 与现有派生属性一致，统计数值和排序保持不变。
8. 将 CI 结果包版本更新为 `v0.10`。
9. 同步 README、flow、flowchart、test 和 update log。

Agent C 必须下载最新 run 的 artifact，并核对：

- manifest 的 `version=v0.10`、`branch`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。
- `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 全部为 `success`。
- `junit.xml` 无 failure，XCTest 不 skipped。
- `xctest.log` 能看到 `JournalEntryTests` 和 `JournalStatisticsTests` 运行并通过。
- `xcodebuild.log` 和 `maccatalyst-build.log` 以 build succeeded 结束，结果包 `.xcresult` 存在。
