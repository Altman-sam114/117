# v0.42（摘要 Markdown 清理单次扫描）

## 版本分配依据

- 当前最新已验收版本为 `v0.41`，本轮继续围绕 Agent X 总目标中的高效写入和长文列表渲染性能做小步优化，因此递增为 `v0.42`。
- 本轮属于低风险模型派生性能优化，不开新大版本。

## 背景

`JournalEntryBodySummary` 和 `JournalSection.excerpt` 会在列表卡片和编辑器小节概览中派生展示摘要。当前实现通过多次 `replacingOccurrences`、`split`、`map(String.init)` 和 `joined` 清理轻量 Markdown 标记。长正文或多小节日记在列表/概览重渲染时会产生多轮中间字符串分配。

## 目标

- 将正文摘要和小节摘要的 Markdown 标记清理改为共享的单次扫描 helper。
- 保持现有摘要语义：正文摘要清理 `#`、`*`、反引号、引用符并按非空行空格拼接；小节摘要额外清理待办标记和列表连字符。
- 增加测试覆盖正文摘要标记清理、空行处理、小节待办标记和普通项目清理。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 `###` 小节识别规则。
- 不改变 Markdown 预览渲染、列表筛选、统计口径、正文编辑、JSON 持久化或 Mac Catalyst 菜单行为。
- 不新增第三方依赖，不调整 deployment target。

## 架构依据

- `JournalEntryBodySummary` 是正文摘要派生边界，应在模型层完成轻量清理。
- `JournalSection.excerpt` 是小节摘要派生边界，应与正文摘要复用相同清理策略并保留小节额外规则。
- 视图只展示摘要结果，不应承担 Markdown 清理逻辑。

## 影响范围和关键文件

- `MDJournal/Models/JournalEntry.swift`：新增 `MarkdownSummaryText` helper，正文摘要和小节摘要复用单次扫描。
- `MDJournalTests/JournalEntryTests.swift`：新增摘要清理语义测试。
- `.github/workflows/ci-results.yml`：更新 `VERSION` 为 `v0.42`。
- `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`：同步当前真实行为。

## 实现步骤

1. 新增私有 `MarkdownSummaryText.plainText(from:removesListMarkers:)`。
2. helper 单次扫描 markdown，跳过指定 Markdown 标记，按行 flush，保持旧的空行丢弃语义。
3. 正文摘要调用 helper，`removesListMarkers` 为 `false`。
4. 小节摘要调用 helper，`removesListMarkers` 为 `true`。
5. 扩展 `JournalEntryTests` 覆盖正文和小节摘要清理结果。
6. 更新 workflow 版本和文档。

## main 直推要求

- 开始前确认当前分支为 `main`，目标远端为 `origin/main`。
- 只提交本轮相关文件。
- commit 后直接 `git push origin main`，触发 `MD Journal CI Results`。
- 不创建候选分支，不创建 PR。

## GitHub Actions 结果包要求

- workflow：`MD Journal CI Results`。
- manifest 中 `version` 必须为 `v0.42`。
- artifact 名称必须对应本轮 `main` commit、run id 和 attempt。
- artifact 至少包含 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log`、`junit.xml` 和可用的 `.xcresult`。

## Agent C 验收要求

Agent C 只验收最新 `origin/main` commit 对应的 run 和 artifact：

- 核对 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。
- 核对 `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 均为 `success`。
- 核对 `junit.xml` 无失败。
- 核对 iOS build、Mac Catalyst build 和 XCTest 日志包含成功标记。
- 核对 `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult`、`MDJournalTests.xcresult` 的 `Info.plist` 可解析。

## 风险和禁止项

- 禁止改变正文摘要和小节摘要的既有展示语义。
- 禁止改变数据持久化格式或绕过 `JournalStore`。
- 禁止把本地命令输出冒充云端 artifact。
- 禁止跳过 Agent C artifact 验收。
- 若云端失败，默认在 `main` 上追加修复 commit 后重新触发 CI，不做回滚式处理。
