# v0.34 / 编辑器头部轻量 metrics

## 版本号

`v0.34`

## 版本分配依据

当前最新正式版本为 `v0.33`。本轮继续沿 `v0.x` 性能优化线推进，目标落在正文编辑热路径：编辑器头部只需要词数和 `###` 小节信息，不需要正文摘要 excerpt，因此可以直接使用 `JournalEntryBodyMetrics`。

## 背景

`EntryEditorView.body` 当前在 `GeometryReader` 内构造 `let bodySummary = entry.bodySummary`。编辑器头部通过 `statPills` 和 `JournalSectionOverview` 只消费 `wordCount`、`sectionCount` 和 `sections`，没有使用 `JournalEntryBodySummary.excerpt`。

`JournalEntryBodySummary` 会先构造 `JournalEntryBodyMetrics`，再额外清理整篇正文生成 excerpt。v0.29 已经拆出轻量 `JournalEntryBodyMetrics`，本轮应让编辑器头部直接消费 metrics，避免正文连续输入时为未使用的摘要做额外字符串处理。

## 目标

- 将 `EntryEditorView` 头部派生从 `entry.bodySummary` 改为 `entry.bodyMetrics`。
- 将 `header` 和 `statPills` 的参数类型从 `JournalEntryBodySummary` 改为 `JournalEntryBodyMetrics`。
- `JournalSectionOverview` 继续消费同一组 `sections`，UI 显示和小节规则不变。
- 更新 GitHub Actions 结果包版本为 `v0.34`，并同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 `JournalEntryBodySummary`、`JournalEntryBodyMetrics`、`JournalSection.extract` 的实现或语义。
- 不改变编辑器 UI 文案、布局、Markdown 预览、写作菜单或片段插入规则。
- 不改变 JSON 持久化格式、`JournalStore` 保存链路或统计口径。
- 不引入第三方依赖。

## 架构依据

- `JournalEntryBodyMetrics` 专门用于不需要摘要时的词数和 `###` 小节轻量派生。
- 视图应只计算当前展示需要的数据；编辑器头部没有 excerpt 展示需求。
- 本轮改动贴合 v0.29 的 metrics 拆分设计，并减少正文编辑时的未用字符串处理。

## 实现步骤

1. 在 `EntryEditorView.body` 中把 `let bodySummary = entry.bodySummary` 改为 `let bodyMetrics = entry.bodyMetrics`。
2. 调用 `header(isWideLayout:bodyMetrics:)`。
3. 将 `header` 参数类型改为 `JournalEntryBodyMetrics`，内部继续把 `bodyMetrics` 传给 `statPills`，并把 `bodyMetrics.sections` 传给 `JournalSectionOverview`。
4. 将 `statPills` 参数类型改为 `JournalEntryBodyMetrics`，继续显示词数、小节数和更新时间。
5. 不改 `JournalSectionOverview`。
6. 将 `.github/workflows/ci-results.yml` 的 `VERSION` 更新为 `v0.34`。
7. 同步 README、测试规范、核心流程、流程图和更新日志。
