# v0.47（Markdown 预览纯文本内联快路径）

## 版本分配依据

- 当前最新已验收版本为 `v0.46`，本轮继续围绕 Agent X 总目标中的 Mac 宽屏写作体验和长文预览性能做小步优化，因此递增为 `v0.47`。
- 本轮只优化 Markdown 预览内联文本渲染路径，不改变用户可见 Markdown 语义，不开新大版本。

## 背景

`MarkdownPreviewView` 在 Mac Catalyst 宽屏写作时会与编辑器并排实时渲染。当前标题、段落、引用和列表项的每个文本片段都会先尝试 `AttributedString(markdown:)`。多数日记正文片段是普通文本，没有加粗、链接、代码或转义等内联 Markdown 语法；这些纯文本片段可以直接构造 `AttributedString(text)`，减少实时预览刷新中的解析尝试。

## 目标

- 为 `MarkdownPreviewView` 增加内联纯文本快路径。
- 没有保守 Markdown 触发字符的文本片段直接返回 `AttributedString(text)`。
- 包含可能触发内联 Markdown 解析的字符时，保留既有 `AttributedString(markdown:)` 和 fallback 行为。
- 增加测试覆盖纯文本跳过解析、加粗/斜体/代码/链接/图片/自动链接/HTML 实体/转义/删除线/表格等触发字符继续走解析路径。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 `MarkdownBlockParser` 的块级解析规则。
- 不改变 `###` 小节分组、有序列表编号、待办渲染、代码块渲染或空内容提示。
- 不做异步解析、缓存、debounce 或后台预览渲染。
- 不改变正文编辑、Markdown 片段插入、JSON 持久化、统计口径或 Mac Catalyst 菜单行为。
- 不新增第三方依赖，不调整 deployment target。
- 不本机运行 app、模拟器、完整 build 或 XCTest；按人工要求最终只用 GitHub Actions artifact 重验证。

## 架构依据

- `MarkdownPreviewView` 是预览渲染唯一入口，适合承载内联文本渲染快路径。
- 块级解析继续由 `MarkdownBlockParser` 负责，预览只消费解析结果并渲染。
- 快路径只减少无样式纯文本片段的 Markdown 解析尝试，不应改变 parser、store 或 model 层。

## 影响范围和关键文件

- `MDJournal/Views/MarkdownPreviewView.swift`：新增 `inlineMarkdown(_:)` 静态 helper 和 `shouldParseInlineMarkdown(_:)` 触发字符判断，调用点复用 helper。
- `MDJournalTests/MarkdownBlockParserTests.swift`：补充内联快路径触发条件测试。
- `.github/workflows/ci-results.yml`：更新 `VERSION` 为 `v0.47`。
- `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`：同步当前真实行为。
- `md/prompt/v0（性能优化）/v0.47（Markdown预览纯文本内联快路径）.md`：保存本轮 Agent A 提示词。

## 实现步骤

1. 将 `MarkdownPreviewView` 的内联文本渲染封装为静态 `inlineMarkdown(_:)`。
2. 新增 `shouldParseInlineMarkdown(_:)`，保守检测 `*`、`_`、反引号、链接/图片括号、自动链接、HTML 实体、转义、删除线和表格等触发字符。
3. 无触发字符时直接返回 `AttributedString(text)`；有触发字符时沿用 `try? AttributedString(markdown:)`，失败则 fallback 到纯文本。
4. 更新预览各调用点使用同一个静态 helper。
5. 增加测试覆盖纯文本和触发字符集合。
6. 更新 workflow 版本和文档。

- 核对 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。
- 核对 `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 均为 `success`。
- 核对 `junit.xml` 无失败。
- 核对 iOS build、Mac Catalyst build 和 XCTest 日志包含成功标记。
- 核对 `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult`、`MDJournalTests.xcresult` 的 `Info.plist` 可解析。

## 风险和禁止项

- 禁止漏掉常见 Markdown 触发字符导致原本可渲染的内联样式被当成纯文本。
- 禁止改变块级 Markdown 解析、`###` 小节识别、有序列表编号、待办渲染或代码块行为。
- 禁止新增缓存状态或把预览派生结果写入模型。
- 禁止跳过 Agent C artifact 验收。
- 若云端失败，默认在 `main` 上追加修复 commit 后重新触发 CI，不做回滚式处理。
