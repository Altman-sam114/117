# v0.39（预览分组判断复用）Agent A 提示词

## 版本号和分配依据

- 版本号：`v0.39`。
- 分配依据：当前最新已验收版本为 `v0.38`，总目标仍是 Mac Catalyst 可用、界面优雅和长文写入高效。本轮继续沿 `v0.x` 做一个小范围 Markdown 预览性能优化。

## 背景

`MarkdownPreviewView` 在单次 `body` 渲染中先构造 `MarkdownParseResult`，随后读取 `document.shouldUseSectionGroups` 来决定 `LazyVStack` 间距和渲染分支。`shouldUseSectionGroups` 当前通过 `sectionGroups.contains { !$0.isIntro }` 派生；同一渲染周期读取两次会重复扫描同一组小节。

## 目标

- 在 `MarkdownPreviewView.body` 中把 `document.shouldUseSectionGroups` 缓存为局部 `let shouldUseSectionGroups`。
- 复用该局部值驱动 `LazyVStack` spacing 和普通/小节分组渲染分支。
- 保持空内容提示、`###` 小节分组条件、块渲染、列表渲染和 UI 间距数值不变。
- 将 GitHub Actions 结果包版本更新为 `v0.39`。
- 同步 README、测试规范、核心流程、流程图、更新日志和本提示词。

## 非目标

- 不修改 `MarkdownBlockParser` 的解析规则或 `MarkdownParseResult` 结构。
- 不修改 `EntryEditorView`、`JournalStore`、本地 JSON 格式或持久化路径。
- 不修改 Markdown 预览 UI 文案、颜色、卡片样式或列表编号显示。
- 不新增第三方依赖，不新增独立 native macOS target。
- 不运行本机完整 build、XCTest、模拟器或 GUI app；人工已要求通过云端回传结果包验证。

## 架构依据

- `MarkdownPreviewView` 只消费解析结果并渲染预览，不负责保存数据。
- `MarkdownBlockParser` 仍是 Markdown 块和 `###` 小节分组的唯一解析边界。
- 本轮只复用同一个解析结果上的布尔派生值，不改变任何数据模型或用户数据。

## 实现步骤

1. 在最新 `origin/main` 的 `main` 分支开始。
2. 在 `MarkdownPreviewView.body` 中解析正文后声明 `let shouldUseSectionGroups = document.shouldUseSectionGroups`。
3. 将 spacing 和 `else if` 分支改为使用该局部值。
4. 将 workflow `VERSION` 更新为 `v0.39`。
5. 同步文档，说明预览分组布尔判断在单次渲染中复用。
