# v0.35 / Mac 专注写作命令

## 版本分配依据

- 当前最新正式版本为 `v0.34`。
- 本轮继续推进总目标“做个 Mac 版、优美写入、高效性能”，沿 `v0.x` 小版本递增为 `v0.35`。
- 本轮属于 Mac Catalyst 写作体验和编辑性能小步优化，放入 `md/prompt/v0（macOS适配）/`。

## 背景

MD Journal 已支持 Mac Catalyst 菜单、“写作”工具栏、Markdown 片段菜单、缩进/反缩进和宽屏预览栏显示/隐藏。当前“显示/隐藏预览”是通用切换命令，但缺少一个明确面向长文输入的“专注写作”入口：在 Mac 宽屏下隐藏预览栏并聚焦正文，在窄屏下回到编辑模式并聚焦正文。

这能让用户更快进入写作状态，也能在宽屏长文输入时减少实时预览栏对 `MarkdownPreviewView` 的解析和渲染压力。

## 目标

- 新增 `EditorWritingCommand.focusWriting`，标题为“专注写作”，使用合适的系统图标。
- 为“专注写作”分配不冲突的 `⌘⌥W` 快捷键。
- Mac Catalyst “写作”菜单遍历命令时能路由该命令。
- `EntryEditorView` 新增专注写作动作：
  - 宽屏时隐藏预览栏。
  - 窄屏时切回编辑模式。
  - 两种布局都聚焦正文。
- Mac Catalyst 写作工具栏新增可见“专注写作”按钮。
- 扩展现有写作命令测试，覆盖命令顺序、快捷键唯一性和新命令无缩进方向。
- 同步 README、核心流程、流程图、测试规范、更新日志和 CI artifact 版本号。

## 非目标

- 不新增独立 native macOS target，继续使用 Mac Catalyst。
- 不修改 `JournalStore`、JSON 持久化、日记模型字段或兼容解码。
- 不修改 Markdown 解析、片段插入、行缩进或回车续写规则。
- 不改现有“显示/隐藏预览”语义。
- 不做本机完整构建、XCTest、模拟器或 GUI 运行；最终以 GitHub Actions artifact 为准。

## 架构依据

- `EditorWritingCommand` 是写作菜单命令集合和快捷键映射边界。
- `MDJournalApp.JournalCommands` 负责把菜单命令路由到 focused scene value。
- `EntryEditorView` 持有编辑/预览模式、预览栏可见性、正文焦点等本地 UI 状态。
- `MarkdownPreviewView` 只消费正文并渲染预览，不负责编辑状态。
- Mac Catalyst 写作命令应复用 focused scene value，不新增全局状态或第二套编辑器入口。

## 实现步骤

1. 在 `EditorWritingCommand` 中加入 `focusWriting`，并更新 `title`、`systemImage`、`indentationDirection`、快捷键映射和测试期望顺序。
2. 在 `MDJournalApp` 中新增 focused scene value：`focusEditorWritingAction`。
3. 更新 `JournalCommands.action(for:)`，把 `focusWriting` 路由到 `focusEditorWritingAction`。
4. 在 `EntryEditorView` 中新增 `focusWriting()`：
   - `mode = .edit`
   - 若 `isWideLayoutActive`，设置 `isPreviewColumnVisible = false`
   - `editorFocused = true`
5. 将 focused scene value 注入 `focusWriting`。
6. 在 Mac Catalyst 工具栏中新增“专注写作”按钮，位置靠近“聚焦正文”。
7. 扩展 `MarkdownSnippetTests`：
   - 写作命令顺序包含 `.focusWriting`。
   - `.focusWriting.indentationDirection == nil`。
   - `EditorWritingCommandShortcut(command: .focusWriting).key == "w"`。
   - 快捷键仍不与 Markdown 片段和 `⌘N` 冲突。
8. 更新 CI 结果包 `VERSION` 为 `v0.35`。
9. 同步 README、核心流程、流程图、测试规范和更新日志。
