# v0.59 / Mac 预览切换状态提示

## 版本号

`v0.59`

## 版本分配依据

当前最新正式版本是 `v0.58`。本轮继续沿 `v0.x` 小版本递增，目标贴合总目标中的“做个 mac 版、优美的写入、高效的性能”，优先处理 Mac Catalyst 写作工具栏中可见动作和 hover 提示的一致性。

## 背景

Mac Catalyst 写作工具栏中的预览切换按钮本身已经会根据状态显示“隐藏预览”“显示预览”或“回到编辑”，但 hover help 仍复用 `EditorWritingCommand.togglePreview.helpText` 的静态“显示/隐藏预览（⌘⌥P）”。桌面用户悬停按钮时看到的动作不够明确，尤其在宽屏双栏、宽屏隐藏预览和窄屏预览模式之间切换时容易产生含混。

## 目标

- 让 Mac Catalyst 写作工具栏预览切换按钮的 hover help 跟随当前状态。
- 保持现有按钮标题状态语义：宽屏预览可见时为“隐藏预览”，宽屏预览隐藏时为“显示预览”，窄屏预览模式时为“回到编辑”。
- 为写作命令 help text 增加纯 Swift helper，使状态标题组合快捷键文案可单元测试。
- 更新 `.github/workflows/ci-results.yml` 的 `VERSION` 为 `v0.59`。
- 同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 `togglePreviewVisibility()` 的状态切换逻辑。
- 不改变“写作”菜单的通用命令标题和快捷键。
- 不改变 Mac Catalyst 宽屏布局、专注写作、正文宽度约束、Markdown 片段、缩进、回车续写、预览解析、统计或本地 JSON 持久化。
- 不新增第三方依赖，不新增独立 native macOS target。
- 不运行本机 app、模拟器、本机构建或 XCTest；最终以 GitHub Actions artifact 验收。

## 架构依据

- `EntryEditorView` 负责当前编辑/预览状态和 Mac Catalyst 写作工具栏，因此 toolbar 的状态化按钮标题仍留在该视图内。
- `EditorWritingCommand` 集中描述写作命令标题、图标、快捷键、预览切换标题和 help text 组合规则，适合承载纯 helper。
- `MarkdownSnippetTests` 已覆盖写作命令元数据、快捷键和 help text，本轮继续在同一测试文件补充状态标题文案断言。

## 实现步骤

1. 在 `EditorWritingCommand` 中新增预览切换标题 helper，统一宽屏/窄屏下“隐藏预览”“显示预览”“回到编辑”的文案规则。
2. 在 `EditorWritingCommand` 中新增 `helpText(title:)`，将传入标题与 `EditorWritingCommandShortcut` 的 `displayText` 组合。
3. 保持现有 `helpText` 属性，内部复用 `helpText(title: title)`，避免影响其他命令。
4. 在 `EntryEditorView` 的预览切换 toolbar 按钮中，将 `.help` 改为使用当前 `previewToggleTitle`。
5. 给同一按钮添加状态化 `accessibilityLabel(previewToggleTitle)`，让辅助功能读到当前动作。
6. 在 `MarkdownSnippetTests` 中断言宽屏/窄屏标题规则，以及“隐藏预览”“显示预览”“回到编辑”均能组合 `⌘⌥P`。
7. 同步 workflow 版本号和文档。
