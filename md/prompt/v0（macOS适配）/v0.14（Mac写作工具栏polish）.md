# v0.14（Mac 写作工具栏 polish）

## 版本号

v0.14

## 版本分配依据

当前最新正式版本为 `v0.13`，已完成列表派生快照性能优化。人工总目标要求继续推进“Mac 版、优美写入、高效性能”。本轮沿 `v0.x` 小版本递增为 `v0.14`，聚焦 Mac Catalyst 写作窗口的桌面工具栏和写作空间 polish。

## 背景

当前 Mac Catalyst 已具备主窗口、统计独立窗口、“日记”菜单、“插入 Markdown”菜单、片段快捷键、右键删除和宽屏编辑/预览双栏。但写作窗口顶部缺少桌面常见的可见写作命令：聚焦正文、快速插入 Markdown、隐藏/显示预览栏。光标/选区插入需要 `TextEditor` 选区桥接或 `UITextView` 包装，风险高于本轮目标，保留为后续独立迭代。

## 目标

- Mac Catalyst 下为 `EntryEditorView` 增加顶部写作工具栏入口。
- 写作工具栏提供“聚焦正文”命令，触发后切回编辑并聚焦正文。
- 写作工具栏提供可见的“插入 Markdown”菜单，复用现有 `MarkdownSnippet` 和追加式插入逻辑。
- 宽屏编辑器支持隐藏/显示右侧预览栏，方便 Mac 写作时获得更宽正文区域。
- 新增 scene-level “写作”菜单，提供聚焦正文和显示/隐藏预览命令。
- 新增可测试的 `EditorWritingCommand` / shortcut 映射，保证写作快捷键不与 Markdown 片段快捷键冲突。
- 保留 iPhone/iPad 原有编辑/预览模式和工具栏行为。
- GitHub Actions 结果包版本更新为 `v0.14`。

## 非目标

- 不实现光标位置插入。
- 不实现选中文本包裹。
- 不引入 `UITextView`、AppKit 或第三方编辑器。
- 不改变 `MarkdownSnippet.markdown` 输出。
- 不改变 `JournalEntry` JSON schema、`JournalStore` 保存链路、Markdown 解析语义或统计口径。
- 不新增 native macOS target，Mac 路径仍是 Mac Catalyst。

## 架构依据

- `MDJournalApp` 是 scene-level commands 入口，适合承载 Mac 菜单和快捷键。
- `EntryEditorView` 是正文编辑、模式切换、Markdown 片段插入和预览布局的 UI 边界。
- `MarkdownSnippet` 和 `MarkdownSnippetCommandShortcut` 已集中管理片段菜单合同。
- 新增 `EditorWritingCommand` 只表达写作菜单命令和快捷键，不接触模型、存储或解析。
- focused scene value 用于让当前编辑器向菜单暴露聚焦正文和切换预览动作。

## 实现步骤

1. 新增 `EditorWritingCommand` 和 `EditorWritingCommandShortcut`，定义聚焦正文和切换预览两个写作命令。
2. 在 `MDJournalApp` 的 commands 中新增“写作”菜单，使用 focused scene value 路由到当前 `EntryEditorView`。
3. 在 `EntryEditorView` 中新增 Mac Catalyst 专属 toolbar item：聚焦正文按钮、插入 Markdown 菜单、显示/隐藏预览按钮。
4. 在 `EntryEditorView` 中新增 `isPreviewColumnVisible` 状态；宽屏时隐藏预览栏后正文编辑区占满剩余空间。
5. 保留窄屏 segmented picker 原行为；如果菜单切换预览在窄屏下触发，仍只在编辑/预览模式之间切换。
6. `insertSnippet(_:)` 继续使用追加式插入，但从 toolbar/menu 触发时仍切回编辑并聚焦正文。
7. 扩展 `MarkdownSnippetTests`，覆盖写作命令元数据、写作快捷键唯一性以及不与片段快捷键冲突。
8. 更新 CI `VERSION` 为 `v0.14`。
9. 同步 README、核心流程、流程图、测试规范和更新日志。
