# v0.9（Mac Catalyst 菜单命令入口）

## 版本号

v0.9

## 版本分配依据

人工总目标要求“做个 mac 版”和“优美的写入高效的性能”。`v0.6` 已启用 Mac Catalyst 构建并补充右键删除与 `⌘N` 新建入口，`v0.7` 和 `v0.8` 已分别优化写入与预览性能。本轮回到 Mac 桌面体验，补齐最小、低风险且符合桌面习惯的菜单命令入口。

## 背景

当前 App 入口只有 `WindowGroup`，没有 scene-level commands。新建入口主要在列表工具栏，统计入口也只在列表工具栏；Mac Catalyst 下虽然已有 `⌘N`，但快捷键挂在按钮上，缺少菜单栏可发现路径。

## 本轮目标

- 为 Mac Catalyst / SwiftUI scene 增加“日记”菜单。
- 菜单提供“新建日记”和“显示统计”两个命令。
- `⌘N` 由菜单命令统一承载，工具栏新建按钮继续保留可见入口但不重复注册快捷键。
- 统计仍复用现有 `StatisticsDashboardView` sheet，不新增独立窗口。
- 将统计 sheet 状态上移到 `ContentView`，方便菜单和列表工具栏共用同一入口。
- 同步 README、测试规范、核心流程、流程图和更新日志中受影响内容。

## 非目标

- 不新增 native macOS target，Mac 路径仍是 Mac Catalyst。
- 不做独立统计窗口、多窗口同步或 AppKit responder-chain 定制。
- 不新增删除快捷键或删除确认流。
- 不改 `JournalStore` 持久化策略、JSON 格式或 Markdown / 统计口径。
- 不引入第三方依赖。

## 架构依据

- `MDJournalApp` 是 scene 和 commands 的入口。
- `ContentView` 拥有 `JournalStore`、选中项和创建/删除动作，适合作为菜单命令的 scene action 路由点。
- `EntryListView` 只展示列表和触发 closure，不持有全局菜单命令状态。
- `StatisticsDashboardView` 继续只负责展示统计。

## 实现步骤

1. 在 `MDJournalApp.swift` 中添加 scene-level `commands` 和“日记”菜单。
2. 使用 SwiftUI focused scene value 将 `ContentView` 的创建日记和显示统计动作暴露给 commands。
3. 在 `ContentView` 中上移 `isShowingStatistics` 状态，新增 `showStatistics()`，并把 closure 传给 `EntryListView`。
4. 在 `EntryListView` 中移除本地统计 sheet 状态，统计工具栏按钮改为调用 `onShowStatistics`，新建按钮保留但移除重复 `⌘N`。
5. 将 CI artifact 版本更新为 `v0.9`。
6. 同步 README、flow、flowchart、test 和 update log。

## 本地轻量检查要求

至少运行：

```sh
git diff --check
git diff --cached --check
plutil -lint MDJournal.xcodeproj/project.pbxproj
xcrun swiftc -parse -parse-as-library $(rg --files -g '*.swift' MDJournal)
ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
```

由于本轮涉及 Mac Catalyst commands，必须本机尝试 Mac Catalyst Debug build。可补 generic iOS Debug build，确保 commands 不破坏 iOS 编译。最终仍以 GitHub Actions artifact 为准。

## main 直推要求

- 开始前同步最新 `origin/main`。
- 不创建候选分支，不创建 PR。
- 只提交本轮相关文件，push 到 `origin/main` 后触发 `MD Journal CI Results`。

## Agent C 云端验收要求

Agent C 必须下载最新 run 的 artifact，并核对：

- manifest 的 `version=v0.9`、`branch`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。
- `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 全部为 `success`。
- `junit.xml` 无 failure，XCTest 不 skipped。
- `maccatalyst-build.log` 以 `** BUILD SUCCEEDED **` 结束，`MDJournalMacCatalyst.xcresult` 存在。
- `xcodebuild.log` 和 `xctest.log` 结尾无失败，结果包 `.xcresult` 存在。

## 风险和禁止项

- 禁止重复注册 `⌘N` 快捷键。
- 禁止把菜单命令变成唯一入口，工具栏按钮必须保留。
- 禁止新增 AppKit responder-chain 复杂逻辑。
- 禁止破坏 iOS / iPadOS 编译。
- 禁止跳过 Agent C artifact 验收。
