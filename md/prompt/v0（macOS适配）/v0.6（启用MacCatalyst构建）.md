# v0.6（启用 Mac Catalyst 构建）

## 版本号

v0.6

## 版本分配依据

本轮由 Agent X 在 `v0.5` Agent X 循环文档基线通过云端 artifact 验收后启动。人工总目标要求“做个 mac 版，优美的写入，高效的性能”。当前工程仍是 iOS-only SwiftUI 项目，且已有多处 UIKit 颜色和 iOS 导航语义。为控制风险并保持 iOS 16.0 与本地 JSON 格式不变，本轮先做 `v0.x` 阶段的 Mac Catalyst 可构建基线，后续再追加桌面写作体验和性能优化轮次。

## 背景

`v0.5` 已把 Agent X 主控循环写入协作规则，并在 `origin/main` 最新 commit `d6e3ade957188e335377bf7d3b41e8ec452a31c5` 上通过 GitHub Actions artifact 验收：run id `28681864701`，run attempt `1`，artifact `mdjournal-ci-v0.4-main-d6e3ade-run28681864701-attempt1`。

当前 Xcode target 只支持 `iphoneos iphonesimulator`，`SUPPORTS_MACCATALYST = NO`。CI 只验证 generic iOS build 和 iOS Simulator XCTest，manifest 版本仍为 `v0.4`，缺少 macOS/Catalyst 构建结果。

## 本轮目标

- 启用现有 `MDJournal` app target 的 Mac Catalyst 支持，保留 iOS 16.0 最低版本和现有 iPhone/iPad 支持。
- 新增 Mac Catalyst Debug build 验证，CI artifact 中记录 Catalyst build outcome、日志路径和 result bundle。
- 修复 Catalyst 编译所需的最小 SwiftUI 平台差异，例如 iOS-only toolbar placement、navigation title display mode 或 UIKit 色彩桥接。
- 增加桌面端基本可用性入口：桌面可通过菜单或 toolbar 使用新建、统计、删除等核心操作，不能只依赖 iOS swipe gesture。
- 不改变 `JournalEntry` JSON 字段、文件名、编码策略或 `JournalStore` 作为唯一持久化入口的架构边界。
- 同步更新 README、测试规范、核心流程、流程图和更新日志中与 Mac Catalyst、CI artifact 相关的部分。

## 非目标

- 不新增独立 native macOS target。
- 不引入第三方依赖。
- 不引入服务器、账号、云同步或跨设备迁移。
- 不改变 `###` 小节语义、Markdown 解析规则或统计口径。
- 不做大规模 UI 重写；桌面视觉和性能深度优化留给后续轮次。
- 不把本机完整模拟器交互验证作为默认完成条件；最终重验证仍以 GitHub Actions artifact 为准。

## 架构依据

- `JournalStore` 仍是唯一日记集合修改与本地 JSON 保存入口。
- `JournalEntry` 仍是本地数据模型和兼容解码边界。
- `ContentView` 仍负责应用级导航、选中项修复和保存错误展示。
- `EntryListView` 负责列表、搜索、分类筛选、统计入口，并应补足桌面删除路径。
- `EntryEditorView` 负责元数据、正文、Markdown 快捷插入和编辑/预览布局；本轮只做 Catalyst 编译和基础桌面适配。
- CI manifest 必须能追溯 `branch`、`commitSha`、`runId`、`runAttempt`、iOS build/test outcome 和 Mac Catalyst build outcome。

## 实现步骤

1. 从最新 `origin/main` 开始，确认当前分支是 `main` 且工作区干净。
2. 在 Xcode project 中启用 app target 的 Mac Catalyst：
   - `SUPPORTED_PLATFORMS` 包含 `macosx`。
   - `SUPPORTS_MACCATALYST = YES`。
   - 如 Xcode 需要，补充合理的 Catalyst bundle id / Mac deployment 设置。
   - 保持测试 target 的 iOS XCTest 基线，不强行把现有 iOS unit test target 改成 macOS 测试。
3. 编译驱动修复 SwiftUI 差异：
   - iOS-only 修饰符用条件编译或小型跨平台 view modifier 包裹。
   - UIKit 颜色桥接集中收敛为 SwiftUI 语义颜色或小型兼容扩展。
   - 桌面端保留 `NavigationSplitView` 的稳定 sidebar-detail 结构。
4. 增加桌面核心操作路径：
   - `MDJournalApp` 或 `ContentView` 可暴露新建日记、打开统计等菜单/command 或 toolbar 路径。
   - 列表行增加 context menu 删除入口；删除仍必须回到 `ContentView.deleteEntry(_:)` 和 `JournalStore.delete(_:)`。
5. 更新 `.github/workflows/ci-results.yml`：
   - `VERSION` 调整为 `v0.6`。
   - 保留现有 static checks、generic iOS Debug build、iOS Simulator XCTest。
   - 新增 Mac Catalyst Debug build，建议 destination 为 `generic/platform=macOS,variant=Mac Catalyst`。
   - artifact 至少新增 `maccatalyst-build.log` 和可用时的 `MDJournalMacCatalyst.xcresult`。
   - manifest 新增 `macCatalystBuildDestination`、`macCatalystResultBundlePath`、`macCatalystBuildLogPath`、`macCatalystBuildOutcome`。
   - JUnit 增加 `maccatalyst-build` testcase，失败数反映 Catalyst outcome。
6. 同步文档：
   - README 写明 Mac Catalyst 支持和验证命令。
   - `md/test/test.md` 增加本地/CI Catalyst 构建检查和 artifact 字段。
   - `md/flow/flow.md` / `flowchart.md` 增加 iOS + Mac Catalyst 构建验证流。
   - `update_log.md` 增加 `v0.6` 记录，并保留 `v0.5` 云端验收信息。

## 本地轻量检查要求

至少运行：

```sh
git diff --check
git diff --cached --check
plutil -lint MDJournal.xcodeproj/project.pbxproj
xcrun swiftc -parse -parse-as-library $(rg --files -g '*.swift' MDJournal)
ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
```

修改 Xcode project 后还应尝试：

```sh
/Applications/Xcode.app/Contents/Developer/usr/bin/xcodebuild -list -project MDJournal.xcodeproj
```

本机如可用，尝试 Mac Catalyst build：

```sh
/Applications/Xcode.app/Contents/Developer/usr/bin/xcodebuild \
  -project MDJournal.xcodeproj \
  -scheme MDJournal \
  -configuration Debug \
  -destination 'generic/platform=macOS,variant=Mac Catalyst' \
  -derivedDataPath /private/tmp/mdjournal-derived-data \
  -resultBundlePath /private/tmp/mdjournal-derived-data/MDJournalMacCatalyst.xcresult \
  CODE_SIGNING_ALLOWED=NO \
  build
```

若本机 Xcode 或 signing 环境导致失败，必须记录关键错误；最终仍以 GitHub Actions artifact 为准。

## main 直推要求

- 本轮默认分支是 `main`，目标远端是 `origin/main`。
- 不创建候选分支，不创建 PR，不执行 PR merge。
- 开始前运行 `git fetch origin` 和 `git pull --ff-only origin main`。
- 只提交本轮相关文件。
- commit 后直接 `git push origin main`，触发 `MD Journal CI Results`。

## Agent C 云端验收要求

Agent C 只验收最新 `origin/main` commit 对应的 GitHub Actions run 和 artifact。必须核对：

- run id、run attempt、branch、commit SHA 与最新 `origin/main` 完全一致。
- `ci-artifact-manifest.json` 中 iOS build/test outcome 和 `macCatalystBuildOutcome` 均为 `success`。
- `junit.xml` 中 `maccatalyst-build` testcase 未失败，XCTest 不是 skipped。
- `static-checks.log`、`xcodebuild.log`、`xctest.log`、`maccatalyst-build.log` 的关键结尾。
- `MDJournal.xcresult`、`MDJournalTests.xcresult`、`MDJournalMacCatalyst.xcresult` 可用时存在。
- artifact 名称应使用 `v0.6`，且下载缓存默认放在 `/private/tmp/mdjournal-c-review-<run_id>/`。

## 风险和禁止项

- 禁止破坏 iOS 16 build、XCTest 和现有本地 JSON 数据。
- 禁止绕过 `JournalStore` 直接改写日记集合或文件。
- 禁止为了 Catalyst 编译删除 Markdown 预览、`###` 小节、分类/心情兼容解码或保存错误展示。
- 禁止把本地未运行或失败的构建伪装成通过。
- 禁止用旧 run、旧 artifact 或 `v0.5` 结果包冒充本轮 `v0.6` 云端验收。
