# v0.32 / Mac 写作缩进菜单工具栏入口

## 版本分配依据

- 当前最新正式版本为 `v0.31`。
- 本轮继续围绕 Agent X 总目标中的“做个 Mac 版、优美写入、高效性能”推进。
- v0.31 已修正 Markdown 反缩进纯规则，本轮把这套已验证规则暴露为 Mac Catalyst 写作菜单和工具栏入口，递增为 `v0.32`。

## 背景

`MarkdownLineIndentation` 已支持 Tab / Shift-Tab 对当前行或多行选区缩进和反缩进，但 Mac Catalyst 可见写作入口目前只有“聚焦正文”和“显示/隐藏预览”。对不熟悉键盘快捷键的用户来说，缩进能力不够可发现；对桌面写作来说，菜单和工具栏入口应该与已有键盘行为共用同一套规则。

## 目标

- `EditorWritingCommand` 新增“增加缩进”和“减少缩进”两个写作命令。
- Mac Catalyst “写作”菜单自动出现新增命令，并通过 focused value 路由到当前编辑器。
- Mac Catalyst 编辑器顶部写作工具栏新增缩进和反缩进按钮。
- 新入口复用 `MarkdownLineIndentation.apply(to:selectedRange:direction:)`，更新 `entry.body` 和 `bodySelectedRange`。
- 扩展写作命令测试，确认命令顺序、元数据、快捷键唯一且不与 Markdown snippet 或 `⌘N` 冲突。
- GitHub Actions 结果包版本更新为 `v0.32`。
- 同步 README、测试规范、核心流程、流程图、更新日志和本提示词。

## 非目标

- 不修改 `MarkdownLineIndentation` 算法。
- 不修改 Tab / Shift-Tab 键盘捕获逻辑、IME marked text 处理或 `MarkdownBodyTextView`。
- 不新增 native macOS target，不修改 deployment target。
- 不修改 JSON 持久化、`JournalStore`、Markdown 预览、片段插入或统计逻辑。
- 不引入第三方依赖。

## 架构依据

- `MarkdownLineIndentation` 是行缩进纯规则层。
- `EntryEditorView` 已持有正文 binding、当前 UTF-16 选区和焦点状态，适合统一执行缩进动作。
- `MDJournalApp` 的 “写作”菜单已通过 `FocusedValues` 连接 scene-level command 和当前编辑器动作，新增缩进 action 应延续这个模式。

## 实现步骤

1. Agent B 开始前同步最新 `origin/main`，确认当前分支为 `main` 且工作区无无关改动。
2. 扩展 `EditorWritingCommand`：
   - 新增 `indentLines` 和 `outdentLines`。
   - 标题使用“增加缩进”和“减少缩进”。
   - 图标使用 `increase.indent` 和 `decrease.indent`。
   - 快捷键使用 `⌘⌥]` 和 `⌘⌥[`，继续避开 `⌘N` 和 Markdown snippet 快捷键。
   - 可新增 `indentationDirection` 只读属性，方便菜单分发和测试。
3. 在 `MDJournalApp` 中新增 focused value：
   - `applyEditorIndentationAction: ((MarkdownLineIndentation.Direction) -> Void)?`
   - “写作”菜单对缩进命令调用该 action，对聚焦和预览命令保持原行为。
4. 在 `EntryEditorView` 中：
   - Mac Catalyst 工具栏新增“增加缩进”和“减少缩进”按钮。
   - 新增 `applyIndentation(_:)`，先 `focusBody()`，再调用 `MarkdownLineIndentation.apply`。
   - 若规则返回 `nil`，保持正文和选区不变。
   - 通过 `.focusedSceneValue(\.applyEditorIndentationAction, applyIndentation)` 暴露给菜单。
5. 更新 `MarkdownSnippetTests`：
   - `EditorWritingCommand.allCases` 期望包含四项。
   - 元数据仍非空。
   - 快捷键唯一、不与 snippet 快捷键冲突、不包含 `n`。
   - 新增断言缩进命令映射到 `.indent` / `.outdent`，非缩进命令为 `nil`。
6. workflow `VERSION` 改为 `v0.32`。
7. 文档只描述新增入口和复用现有规则，不宣称本地 UI 已人工交互运行。


## 风险和禁止项

- 风险：工具栏按钮增多，需要保持紧凑、中文工具型风格；本轮只加两个 icon+label 按钮并复用现有样式。
- 风险：快捷键必须避免与 snippet 和 `⌘N` 冲突。
- 禁止修改缩进算法、持久化格式、`JournalStore`、Markdown 预览解析、片段插入或 `###` 小节识别。
- 禁止用本地输出或旧 artifact 冒充本轮云端验收。
