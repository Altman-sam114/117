# v0.17（Mac Catalyst 一键构建运行入口）

## 版本号

v0.17

## 版本分配依据

当前最新正式版本为 `v0.16`，已优化 `JournalStore.update(_:)` 的更新排序成本。人工总目标要求继续推进“Mac 版、优美写入、高效性能”。项目已经支持 Mac Catalyst 构建和桌面菜单，但本地运行仍依赖人工打开 Xcode 或手写 `xcodebuild` 命令；Codex 桌面侧也没有项目内 `Run` action。本轮沿 `v0.x` 小版本递增为 `v0.17`，聚焦 Mac 版本地交付和运行效率。

## 背景

当前仓库是 Xcode project 形态，只有 `MDJournal.xcodeproj` 和 `MDJournal` scheme。Mac 版本路线是 Mac Catalyst，不新增 native macOS target。`xcodebuild -list` 能识别 `MDJournal` scheme，但默认会尝试写入用户 home 下的 DerivedData 并产生权限噪声；现有验证命令已经使用 `/private/tmp` 下的 DerivedData 避开这个问题。

`build-macos-apps:build-run-debug` 要求项目存在一个稳定的 `script/build_and_run.sh`，负责停止旧进程、构建 Mac Catalyst app、运行新 app，并提供 `--debug`、`--logs`、`--telemetry`、`--verify` 模式。同时 `.codex/environments/environment.toml` 应暴露 Codex app 的 `Run` action。

## 目标

- 新增项目内 `script/build_and_run.sh`，作为 Mac Catalyst 的唯一一键构建/运行入口。
- 脚本默认模式执行：停止已有 `MDJournal` 进程、构建 Mac Catalyst Debug app、用 `/usr/bin/open -n` 启动最新构建产物。
- 支持 `--verify`：启动后用 `pgrep -x MDJournal` 确认进程存在。
- 支持 `--debug`：构建后用 `lldb -- <app binary>` 进入调试。
- 支持 `--logs`：启动 app 后用 unified logging 过滤 `process == "MDJournal"`。
- 支持 `--telemetry`：启动 app 后用 bundle id 过滤 unified logging。
- 显式使用 `/Applications/Xcode.app/Contents/Developer/usr/bin/xcodebuild` 和 `/private/tmp/mdjournal-build-and-run`，避免默认写入 home DerivedData。
- 新增 `.codex/environments/environment.toml`，暴露 `Run` action 指向 `./script/build_and_run.sh`。
- 更新 README、测试规范、核心流程、流程图和更新日志。
- GitHub Actions 结果包版本更新为 `v0.17`。

## 非目标

- 不新增 native macOS target，仍使用现有 Mac Catalyst target。
- 不修改 Swift 业务逻辑、UI 布局、菜单命令或本地 JSON 数据路径。
- 不引入第三方依赖、包管理器、服务器、账号或云同步。
- 不改变 CI 的核心阶段，只更新 artifact 版本号。
- 不把构建产物、DerivedData 或运行日志提交进仓库。

## 架构依据

- `MDJournal.xcodeproj` / `MDJournal` scheme 是当前唯一 app-producing Xcode 入口。
- Mac Catalyst bundle id 当前是 `com.codex.mdjournal.mac`，用于 telemetry 日志过滤。
- `script/build_and_run.sh` 属于项目运行辅助，不属于 Swift app source。
- `.codex/environments/environment.toml` 属于 Codex 桌面环境配置，不属于 app 代码。
- 云端仍由 `.github/workflows/ci-results.yml` 负责 generic iOS build、Mac Catalyst build 和 XCTest artifact。

## 实现步骤

1. 新增 `script/build_and_run.sh`，使用 `set -euo pipefail`。
2. 定义项目常量：project、scheme、configuration、app name、bundle id、DerivedData 路径、Mac Catalyst destination 和 app bundle 路径。
3. 先 `pkill -x MDJournal`，忽略不存在进程的错误。
4. 用 `xcodebuild -project MDJournal.xcodeproj -scheme MDJournal -configuration Debug -destination 'generic/platform=macOS,variant=Mac Catalyst' -derivedDataPath /private/tmp/mdjournal-build-and-run CODE_SIGNING_ALLOWED=NO build` 构建。
5. 构建后确认 `MDJournal.app` 和 `Contents/MacOS/MDJournal` 存在。
6. 根据模式执行：
   - `run`：`/usr/bin/open -n "$APP_BUNDLE"`。
   - `--verify`：open 后 sleep，再 `pgrep -x MDJournal`。
   - `--debug`：`lldb -- "$APP_BINARY"`。
   - `--logs`：open 后 `log stream --info --style compact --predicate 'process == "MDJournal"'`。
   - `--telemetry`：open 后 `log stream --info --style compact --predicate 'subsystem == "com.codex.mdjournal.mac"'`。
7. 新增 `.codex/environments/environment.toml`，使用 canonical Run action 形态。
8. 给脚本加执行位。
9. 更新 CI `VERSION` 为 `v0.17`。
10. 同步 README、测试规范、核心流程、流程图和更新日志。

## 验证要求

本轮人工要求以云端回传测试为准，不在本机执行 build、run 或 XCTest。Agent B 提交前只做不启动 app 的静态轻量检查：

```sh
bash -n script/build_and_run.sh
test -x script/build_and_run.sh
git diff --check
plutil -lint MDJournal.xcodeproj/project.pbxproj
ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
xcrun swiftc -parse -parse-as-library $(rg --files -g '*.swift' MDJournal)
```

不默认执行 `./script/build_and_run.sh --verify`；该命令会启动本机 GUI app，只有人工明确要求本机运行时才使用。Mac Catalyst build、XCTest 和 manifest 结论必须来自 GitHub Actions artifact。

- workflow：`MD Journal CI Results`。
- artifact 名称必须使用 `mdjournal-ci-v0.17-main-<short_sha>-run<run_id>-attempt<run_attempt>`。
- manifest 必须记录 `version=v0.17`、`branch=main`、本轮 `commitSha`、`runId`、`runAttempt`、`macCatalystBuildOutcome`、`buildOutcome` 和 `testOutcome`。

## Agent C 验收要求

Agent C 必须下载最新 `origin/main` 对应 run 的未加密 artifact 到 `/private/tmp/mdjournal-c-review-<run_id>/`，并核对：

- manifest 的 `version`、`branch`、`commitSha`、`runId`、`runAttempt` 和当前最新 `origin/main` 完全一致。
- `junit.xml` 失败数为 0，且 XCTest 没有被跳过。
- `static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log` 与 manifest outcome 一致。
- `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult` 和 `MDJournalTests.xcresult` 存在。
- 文档已记录 v0.17 的本地 run script、Codex Run action、验证结果、artifact 名称和遗留事项。
