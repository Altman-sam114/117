# v0.41（Mac 写作工具栏快捷键提示）

## 版本分配依据

- 当前最新已验收版本为 `v0.40`，本轮继续围绕 Agent X 总目标中的 Mac 版写作效率和桌面体验打磨，因此递增为 `v0.41`。
- 本轮只做低风险 Mac Catalyst 工具栏提示增强，不开新大版本。

## 背景

Mac Catalyst 已有“写作”菜单和写作工具栏，菜单项通过 `EditorWritingCommandShortcut` 暴露 `⌘⌥` 快捷键；但工具栏按钮的 hover help 只显示命令标题，用户需要进入菜单才能发现快捷键。桌面写作工具应让常用命令的快捷键更容易被发现。

## 目标

- 给 Mac Catalyst 写作工具栏按钮的 hover help 补充对应 `⌘⌥` 快捷键。
- 让 help 文案复用 `EditorWritingCommand` / `EditorWritingCommandShortcut` 的命令模型，避免视图硬编码快捷键。
- 增加测试覆盖写作命令 help 文案包含命令标题和快捷键展示。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不新增或修改写作命令。
- 不新增或修改快捷键绑定。
- 不改变 Markdown 插入、缩进、预览显示、专注写作、正文输入、JSON 持久化或统计口径。
- 不新增第三方依赖，不调整 deployment target。

## 架构依据

- `EditorWritingCommand` 是写作命令的标题、图标和缩进方向来源。
- `EditorWritingCommandShortcut` 是写作命令键盘快捷键来源。
- `EntryEditorView` 的 Mac Catalyst 工具栏只应消费命令模型，不应复制快捷键规则。

## 影响范围和关键文件

- `MDJournal/Utilities/EditorWritingCommand.swift`：新增快捷键展示和 help 文案。
- `MDJournal/Views/EntryEditorView.swift`：Mac Catalyst 写作工具栏 `.help` 使用命令 help 文案。
- `MDJournalTests/MarkdownSnippetTests.swift`：扩展写作命令元数据和快捷键展示测试。
- `.github/workflows/ci-results.yml`：更新 `VERSION` 为 `v0.41`。
- `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`：同步当前真实行为。

## 实现步骤

1. 在 `EditorWritingCommandShortcut` 中增加 `displayText`，格式为 `⌘⌥` 加大写 key。
2. 在 `EditorWritingCommand` 中增加 `helpText`，由命令标题和 shortcut display text 组成。
3. 将 `EntryEditorView` 中 Mac Catalyst 写作工具栏按钮的 `.help(command.title)` 改为 `.help(command.helpText)`。
4. 在 `MarkdownSnippetTests` 中断言每个写作命令 help 文案包含命令标题和对应快捷键展示，并覆盖关键命令的展示文案。
5. 更新 workflow 版本和文档。

## main 直推要求

- 开始前确认当前分支为 `main`，目标远端为 `origin/main`。
- 只提交本轮相关文件。
- commit 后直接 `git push origin main`，触发 `MD Journal CI Results`。
- 不创建候选分支，不创建 PR。

## GitHub Actions 结果包要求

- workflow：`MD Journal CI Results`。
- manifest 中 `version` 必须为 `v0.41`。
- artifact 名称必须对应本轮 `main` commit、run id 和 attempt。
- artifact 至少包含 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log`、`junit.xml` 和可用的 `.xcresult`。

## Agent C 验收要求

Agent C 只验收最新 `origin/main` commit 对应的 run 和 artifact：

- 核对 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。
- 核对 `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 均为 `success`。
- 核对 `junit.xml` 无失败。
- 核对 iOS build、Mac Catalyst build 和 XCTest 日志包含成功标记。
- 核对 `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult`、`MDJournalTests.xcresult` 的 `Info.plist` 可解析。

## 风险和禁止项

- 禁止改变已有快捷键或写作命令顺序。
- 禁止把本地命令输出冒充云端 artifact。
- 禁止跳过 Agent C artifact 验收。
- 若云端失败，默认在 `main` 上追加修复 commit 后重新触发 CI，不做回滚式处理。
