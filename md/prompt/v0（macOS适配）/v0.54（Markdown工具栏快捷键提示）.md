# v0.54（Markdown 工具栏快捷键提示）

## 版本号

`v0.54`

## 版本分配依据

- 当前最新已验收版本为 `v0.53`，本轮继续围绕 Agent X 总目标中的 Mac 版写作效率和桌面体验推进。
- 本轮只补齐 Markdown 正文工具栏 hover 提示中的键盘快捷键，不改变插入规则或业务状态流，因此沿 `v0.x` 小版本递增为 `v0.54`。

## 背景

Mac Catalyst 下，“写作”工具栏按钮已经通过 `EditorWritingCommand.helpText` 显示命令标题和 `⌘⌥` 快捷键，提升桌面快捷键可发现性。但正文 Markdown 图标工具栏目前 `.help(snippet.title)` 只显示片段名，而同一套 Markdown 片段已经在“插入 Markdown”菜单中绑定了 `MarkdownSnippetCommandShortcut`。

为保持 Mac 桌面写作入口一致，本轮让 Markdown 工具栏 hover 提示复用同一快捷键映射，例如“加粗（⌘⌥B）”。

## 目标

- 为 `MarkdownSnippetCommandShortcut` 增加面向 UI 提示的 `displayText`。
- 为 `MarkdownSnippet` 增加 `helpText`，格式为 `标题（快捷键）`。
- 将 `MarkdownToolbar` 的 `.help` 从片段标题改为 `snippet.helpText`。
- 补充 `MarkdownSnippetTests` 覆盖片段 help text 包含标题和快捷键显示文案。
- 同步 workflow 版本、README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不修改 Markdown 片段插入结果、快捷键按键、快捷键 modifiers 或菜单命令。
- 不改变正文光标/选区、Markdown 续写、缩进、预览、统计或本地 JSON 持久化。
- 不引入第三方依赖，不修改 deployment target。
- 不运行本机完整 build、XCTest、模拟器或 app；按人工要求最终只用 GitHub Actions artifact 重验证。

## 架构依据

- `MarkdownSnippet` 是 Markdown 片段元数据边界，适合承载统一的标题、图标、Markdown 和工具提示文案。
- `MarkdownSnippetCommandShortcut` 是片段菜单快捷键映射边界，工具栏提示应复用它而不是复制快捷键字符串。
- `MarkdownToolbar` 只负责渲染按钮并调用 `onInsert`，不负责定义快捷键。

## 实现步骤

1. 确认当前分支为 `main` 且同步最新 `origin/main`。
2. 在 `MarkdownSnippetCommandShortcut` 增加 `displayText`。
3. 在 `MarkdownSnippet` 增加 `helpText`，复用 `MarkdownSnippetCommandShortcut(snippet: self).displayText`。
4. 将 `MarkdownToolbar` 的 `.help(snippet.title)` 改为 `.help(snippet.helpText)`，保留 `accessibilityLabel(snippet.title)`。
5. 在 `MarkdownSnippetTests` 中验证 help text 包含标题和快捷键显示文案，并补充代表性快捷键断言。
6. 更新 workflow 版本和文档。

## main 直推要求

- 默认分支必须是 `main`，目标远端必须是 `origin/main`。
- 只提交本轮相关文件。
- commit 后直接 `git push origin main`，触发 `MD Journal CI Results`。
- 不创建候选分支，不创建 PR。

## GitHub Actions 结果包要求

- workflow：`MD Journal CI Results`。
- manifest 中 `version` 必须为 `v0.54`。
- artifact 名称必须对应本轮最新 `main` commit、run id 和 attempt。
- artifact 至少包含 manifest、failure summary、主日志、JUnit 和可用的 `.xcresult`。

## Agent C 验收要求

Agent C 只验收最新 `origin/main` commit 对应的 run 和 artifact：

- 核对 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。
- 核对 `staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome`、`testOutcome` 均为 `success`。
- 核对 `junit.xml` 显式显示 `failures=0`、`errors=0`。
- 核对 iOS build、Mac Catalyst build 和 XCTest 日志包含成功标记。
- 核对 `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult`、`MDJournalTests.xcresult` 的 `Info.plist` 可解析。

## 风险和禁止项

- 禁止改动快捷键映射或菜单命令行为。
- 禁止改变 `MarkdownToolbar` 的 `onInsert` 调用、按钮可访问性标题或片段插入结果。
- 禁止跳过 Agent C artifact 验收。
- 若云端失败，默认在 `main` 上追加修复 commit 后重新触发 CI，不做回滚式处理。
