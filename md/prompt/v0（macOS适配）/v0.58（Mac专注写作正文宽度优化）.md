# v0.58 / Mac 专注写作正文宽度优化

## 版本号

`v0.58`

## 版本分配依据

当前最新正式版本是 `v0.57`。本轮继续沿 `v0.x` 小版本递增，目标贴合总目标中的“做个 mac 版、优美的写入、高效的性能”。

## 背景

Mac Catalyst 宽屏下，编辑器默认左右分栏；触发“专注写作”或隐藏预览栏后，正文编辑栏会占满整个宽窗口。超宽窗口里正文行长过长，降低长文输入舒适度，也容易让输入视线横向漂移。

## 目标

- 在宽屏且预览栏隐藏时，让正文输入区居中并限制到舒适最大宽度。
- 保持编辑器工具栏、顶部元数据区和宽屏容器全宽，避免破坏现有桌面工具入口。
- 保留预览栏可见时的双栏行为。
- 保留窄屏 segmented 编辑/预览行为。
- 更新 `.github/workflows/ci-results.yml` 的 `VERSION` 为 `v0.58`。
- 同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不新增独立 native macOS target，继续使用 Mac Catalyst。
- 不改变 `JournalStore`、本地 JSON、Markdown 解析、Markdown 片段、Tab / Shift-Tab 缩进、回车续写或统计口径。
- 不改变 Mac 菜单命令、快捷键、focused scene value 或 toolbar 按钮顺序。
- 不运行本机 app、模拟器、本机构建或 XCTest；最终以 GitHub Actions artifact 验收。

## 架构依据

- `EntryEditorView` 负责编辑/预览布局和 Mac Catalyst 写作工具栏。
- `MarkdownBodyTextView` 只负责正文 UIKit bridge，不应承载外层布局策略。
- 宽屏阈值仍沿用当前 `820` pt。

## 影响范围和关键文件

- `MDJournal/Views/EntryEditorView.swift`
- `.github/workflows/ci-results.yml`
- `README.md`
- `md/test/test.md`
- `md/flow/flow.md`
- `md/flow/flowchart.md`
- `update_log.md`

## 实现步骤

1. 在 `EntryEditorView` 中引入私有常量，例如 `focusedWritingMaxWidth`。
2. 拆分宽屏编辑列内容，使预览栏可见时保持原双栏等宽。
3. 当 `isPreviewColumnVisible == false` 时，用 `HStack` / `Spacer` 或等价 SwiftUI 布局把正文输入区居中，并限制最大宽度。
4. 不把宽度限制应用到窄屏 `compactEditor`。
5. 不改变 `focusWriting()` 和 `togglePreviewVisibility()` 的状态语义。
