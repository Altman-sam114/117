# v0.11（Mac Catalyst 统计独立窗口）

## 版本分配依据

当前最新正式版本为 `v0.10`，已完成正文派生与统计计算单次化。人工总目标要求继续优化 Mac 版、写作体验与性能；本轮沿 `v0.x` 小版本递增为 `v0.11`，聚焦 Mac Catalyst 桌面体验，不改变数据模型。

## 背景

项目已支持 Mac Catalyst build，v0.9 增加了“日记”菜单和“显示统计”命令，v0.10 已降低统计计算重复开销。当前统计看板在 iOS/iPadOS 和 Mac Catalyst 下都以 sheet 展示，桌面体验仍偏移动端。

Agent X 并发勘察后得到三个候选：

- Mac Catalyst 统计独立窗口。
- Markdown 片段键盘命令。
- 列表搜索和分类筛选派生单次化。

本轮选择第一个候选，因为它最直接推进 Mac 版体验，并且能在不改 JSON schema、不改统计口径的前提下独立验收。

## 目标

- Mac Catalyst 下，“日记”菜单和列表工具栏的统计入口打开独立统计窗口。
- iOS/iPadOS 保持现有统计 sheet 行为。
- 统计窗口复用同一个 `JournalStore` 的当前 `entries`，不新增第二套数据加载或保存路径。
- 独立窗口中的统计看板去掉移动端 sheet 关闭按钮，依赖桌面窗口 chrome 关闭。
- GitHub Actions artifact 版本更新为 `v0.11`。
- 同步 README、核心流程、流程图、测试规范和更新日志。

## 非目标

- 不新增 native macOS target。
- 不改变本地 JSON 文件名、字段、解码兼容或保存策略。
- 不改变 `JournalStatistics` 指标口径。
- 不改变 Markdown 解析、`###` 小节语义或预览样式。
- 不加入统计快捷键，避免和保存、系统命令或未来写作命令冲突。
- 不实现多窗口编辑器或多文档模型。

## 架构依据

- `JournalStore` 仍是唯一日记集合修改与本地持久化入口。
- `ContentView` 保持应用级导航、选中项修复和保存错误展示；选中项不提升到 App 级，避免多个窗口共享编辑选择。
- App 级可以持有共享 `JournalStore`，主窗口与统计窗口只读/写同一 store 状态。
- `StatisticsDashboardView` 只消费 `[JournalEntry]` 并渲染统计，不负责持久化。
- Mac Catalyst 使用 SwiftUI scene/window API；iOS 路径继续使用 sheet。

## 实现步骤

1. 在 `MDJournalApp` 中把 `JournalStore` 上移为 App 级 `@StateObject`。
2. 主 `WindowGroup` 继续创建 `ContentView`，但改为注入共享 store。
3. 仅在 `targetEnvironment(macCatalyst)` 下注册统计 `WindowGroup`，内容为 `StatisticsDashboardView(entries: store.entries, showsCloseButton: false)`。
4. `ContentView` 改为 `@ObservedObject var store`，保留本地 `selectedEntryID`、统计 sheet 状态和 scene phase flush。
5. `ContentView.showStatistics()` 在 Mac Catalyst 下调用 `openWindow(id:)`，非 Catalyst 下保持 `isShowingStatistics = true`。
6. `StatisticsDashboardView` 增加 `showsCloseButton` 参数，默认 `true` 保持 sheet 行为；统计窗口传 `false`。
7. 更新 CI `VERSION` 为 `v0.11`。
8. 同步文档中统计入口、Mac Catalyst 行为、artifact 命名和本轮记录。

## 本地轻量检查要求

Agent B 提交前至少运行：

```sh
git diff --check
plutil -lint MDJournal.xcodeproj/project.pbxproj
ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
xcrun swiftc -parse -parse-as-library $(rg --files -g '*.swift' MDJournal)
/Applications/Xcode.app/Contents/Developer/usr/bin/xcodebuild -project MDJournal.xcodeproj -scheme MDJournal -configuration Debug -destination 'generic/platform=macOS,variant=Mac Catalyst' -derivedDataPath /private/tmp/mdjournal-derived-data -resultBundlePath /private/tmp/mdjournal-derived-data/MDJournalMacCatalyst.xcresult CODE_SIGNING_ALLOWED=NO build
```

如修改导致 generic iOS build 风险，也运行 generic iOS Debug build。当前机器 CoreSimulator 服务不可用时，不强制本机 XCTest，最终以 GitHub Actions artifact 为准。

- 结果包至少包含 manifest、JUnit、`static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log`、失败摘要和可用 `.xcresult`。

## Agent C 验收要求

Agent C 必须下载最新 `origin/main` 对应 run 的未加密 artifact 到 `/private/tmp/mdjournal-c-review-<run_id>/`，并核对：

- manifest 的 `version`、`branch`、`commitSha`、`runId`、`runAttempt` 和当前最新 `origin/main` 完全一致。
- `junit.xml` 失败数为 0，且 XCTest 没有被跳过。
- `static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log` 与 manifest outcome 一致。
- `maccatalyst-build.log` 以 `** BUILD SUCCEEDED **` 结束，`MDJournalMacCatalyst.xcresult` 存在。
- 文档已记录 v0.11 行为、验证结果、artifact 名称和遗留事项。

## 风险和禁止项

- 禁止新增第二个独立 `JournalStore` 加载同一 JSON，以免统计窗口与主窗口状态不同步。
- 禁止把 `selectedEntryID` 提升为 App 全局状态。
- 禁止改变 JSON schema、统计口径或 Markdown 解析语义。
- 禁止用旧 run 或旧 artifact 冒充本轮 v0.11 云端验收。
- 如果 `openWindow` 或 Catalyst `WindowGroup(id:)` 在当前 deployment target 下无法编译，必须退回可构建的小方案或暂停记录阻塞，不能伪装通过。
