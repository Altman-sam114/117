# v0.23（Markdown 有序列表回车续写）

## 版本号

`v0.23`

## 版本分配依据

当前最新正式版本为 `v0.22`。本轮继续沿 `v0.x` 小步优化，聚焦 Markdown 正文写作效率和 Mac Catalyst / 硬件键盘连续输入体验，不改变数据模型、持久化、预览语法或统计看板。

## 背景

`MarkdownBodyTextView` 已把正文回车输入集中路由到 `MarkdownLineContinuation`，当前已支持无序列表、待办和引用的回车续写。写计划、复盘步骤或操作清单时，有序列表是常见 Markdown 结构，但当前输入 `1. 今天` 后回车仍只插入普通换行，用户需要手动输入 `2. `。这会降低 Mac Catalyst 和硬件键盘场景下的连续写作效率。

## 目标

- 扩展 `MarkdownLineContinuation`，支持 Markdown 有序列表行回车续写。
- 支持行首缩进后的 `数字. ` 前缀，例如 `1. 第一项` 回车后生成 `2. `。
- 非空有序列表项按回车时延续相同缩进，并把编号递增 1。
- 空有序列表项按回车时删除当前缩进和有序列表前缀，退出列表。
- 光标在有序列表行中间时按回车拆分当前行，并让下一行继续有序列表。
- 编号无法安全解析或已达到整数上限时不接管输入，走系统默认行为。
- 保持已有无序列表、待办、引用、IME marked text、Tab / Shift-Tab 和片段插入行为不变。
- 新增或扩展 `MarkdownLineContinuationTests`，覆盖有序列表续写、空项退出、缩进、行中拆分、fenced code、非折叠选区、普通输入和 UTF-16/emoji 光标边界。
- 将 CI artifact 版本更新为 `v0.23`，并同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不支持 `1) ` 变体；本轮只支持 Markdown 常见且范围更明确的 `数字. `。
- 不做整段自动重编号，不处理删除中间项后的后续编号调整。
- 不新增有序列表工具栏按钮、菜单项或快捷键。
- 不新增有序列表预览语法，`MarkdownBlockParser` 和 `MarkdownPreviewView` 行为保持不变。
- 不改变 `MarkdownSnippet` 文案、快捷键或工具栏顺序。
- 不改变 `JournalStore` 保存、排序、节流或 JSON 持久化。
- 不引入第三方依赖，不新增 native macOS target。

## 架构依据

- `MarkdownBodyTextView` 只负责 UIKit bridge 和输入事件入口，不承载 Markdown 字符串规则。
- `MarkdownLineContinuation` 属于规则层，消费正文、UTF-16 `NSRange` 和 replacement text，输出新正文与新选区。
- 有序列表续写属于纯文本输入规则，应在规则层实现并用 XCTest 覆盖。
- Mac Catalyst 仍走现有 target 和 GitHub Actions Mac Catalyst build 验证。

## 实现步骤

1. 在 `MarkdownLineContinuation` 中扩展行前缀识别：
   - 保留现有无序列表、待办和引用识别、续写、退出逻辑。
   - 增加有序列表前缀识别，允许行首空格或 tab 缩进。
   - 只识别 `数字. `，例如 `1. `、`12. `。
   - 非空项续写时生成同缩进的下一编号前缀。
   - 空项退出时删除当前行缩进和有序列表前缀。
   - 编号无法解析或递增溢出时返回空结果，让系统默认输入处理。
2. 保持 `MarkdownBodyTextView.Coordinator.shouldChangeTextIn` 的入口不扩大：
   - marked text 存在时继续走系统默认行为。
   - 只有纯规则返回结果时才手动写回 text view、正文 binding 和 selectedRange binding。
3. 扩展 `MarkdownLineContinuationTests`：
   - 非空有序列表行末尾续写并递增编号。
   - 空有序列表行退出。
   - 光标在有序列表行中间拆分。
   - 缩进有序列表保留缩进。
   - fenced code block 内不触发。
   - 非折叠选区和非回车输入不触发。
   - emoji / UTF-16 光标保持正确。
   - 已有无序列表、待办和引用用例继续保留。
4. 更新 workflow `VERSION: v0.23`。
5. 同步 README、测试规范、核心流程、流程图和更新日志。
