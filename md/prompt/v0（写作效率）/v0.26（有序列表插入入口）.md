# v0.26（有序列表插入入口）

## 版本号

`v0.26`

## 版本分配依据

当前最新正式版本为 `v0.25`。v0.23 已支持有序列表回车续写，v0.24 已支持有序列表预览，但正文工具栏、Mac Catalyst “插入 Markdown”菜单和键盘快捷键仍没有有序列表插入入口。本轮沿 `v0.x` 小步补齐写作链路，让 Mac Catalyst 和硬件键盘用户能从可见 UI 和菜单直接插入有序列表。

## 背景

当前 `MarkdownSnippet` 支持小节、加粗、斜体、引用、无序列表、待办、代码和分割线。用户写步骤、计划或操作清单时，有序列表是常见结构，但必须手动输入 `1. ` 才能进入 v0.23 的续写规则。补齐片段入口能让“插入 -> 回车续写 -> 预览渲染”形成完整闭环。

## 目标

- 在 `MarkdownSnippet` 中新增有序列表片段，标题使用“有序列表”，默认 Markdown 为 `1. `。
- 正文工具栏、Mac Catalyst “插入 Markdown”菜单和键盘快捷键自动包含该片段。
- 空选区插入 `1. `，光标停在前缀后，后续回车续写由 `MarkdownLineContinuation` 负责。
- 选中多行时按 `1. `、`2. `、`3. ` 递增编号逐行转换，保留尾随换行处理方式。
- 保持现有无序列表、待办、引用、代码、分割线和包裹类片段行为不变。
- 更新 `MarkdownSnippetTests` 覆盖元数据、顺序、快捷键唯一性、markdown contract 和有序列表选区插入。
- 将 CI artifact 版本更新为 `v0.26`，并同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不改变 v0.23 的回车续写规则。
- 不改变 v0.24 的有序列表预览解析和渲染。
- 不支持 `1) ` 变体、嵌套有序列表或自动重编号已有正文。
- 不新增 native macOS target，不引入第三方 Markdown renderer。
- 不改变 `JournalStore` 保存、排序、节流或 JSON 持久化。

## 架构依据

- `MarkdownSnippet` 是工具栏和菜单共用的片段元数据来源。
- `MarkdownSnippetInsertion` 是光标/选区插入规则边界；有序列表选区编号应在这里作为纯规则实现并测试。
- `EntryEditorView`、`MarkdownToolbar` 和 `MDJournalApp` 已通过 `MarkdownSnippet.allCases` 自动生成 UI 和菜单，本轮不应复制第二套入口。
- 云端 artifact 必须证明 Swift parse、iOS build、Mac Catalyst build 和 XCTest 都通过。

## 实现步骤

1. `MarkdownSnippet` 新增 `orderedList` case：
   - 标题：“有序列表”。
   - SF Symbol 使用 `list.number`。
   - markdown 为 `1. `。
2. `MarkdownSnippetInsertion` 支持有序列表：
   - 空选区按 `1. ` 插入，光标停在末尾。
   - 非空选区逐行编号，编号从 1 开始递增。
   - 保留选区尾随换行的处理方式，不额外给尾随空行编号。
3. `MarkdownSnippetCommandShortcut` 为有序列表分配不冲突快捷键：
   - 避开 `⌘N` 和已有 `⌘⌥` 片段 / 写作命令。
   - 建议使用 `⌘⌥O`，测试固定唯一性。
4. 扩展 `MarkdownSnippetTests`：
   - `allCases.count`、顺序和 markdown contract 更新。
   - 快捷键唯一性继续通过。
   - 选中多行插入有序列表会得到 `1. 第一行\n2. 第二行`。
   - 尾随换行选区不产生额外空编号。
5. 更新 `.github/workflows/ci-results.yml` 的 `VERSION` 为 `v0.26`。
6. 同步 README、测试规范、核心流程、流程图和更新日志。
