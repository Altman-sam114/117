# v0.18（Markdown 列表回车续写）

## 版本号

v0.18

## 版本分配依据

当前最新正式版本为 `v0.17`，已增加 Mac Catalyst 一键构建/运行入口和 Codex `Run` action。人工总目标继续要求推进“Mac 版、优美写入、高效性能”，且明确要求验证以 GitHub Actions 回传结果包为准，不走本机运行测试。本轮沿 `v0.x` 小版本递增为 `v0.18`，聚焦正文写作手感中的一个小闭环：Markdown 列表和待办的回车续写。

## 背景

`MarkdownBodyTextView` 当前只负责 `UITextView` 正文绑定、光标/选区同步和焦点同步。用户在 `- `、`- [ ] ` 这类 Markdown 列表中按回车时，系统默认只插入换行，不会自动带出下一行列表前缀；在空列表项按回车也不会退出列表。这会打断 Mac Catalyst 和 iPad 键盘写作节奏。

## 目标

- 新增可测试的纯规则层 `MarkdownLineContinuation`，根据正文、当前 UTF-16 选区和 replacement text 判断是否需要接管回车。
- 在普通列表项按回车时，自动插入下一行同缩进的列表前缀，例如 `- 内容` -> `- 内容\n- `。
- 在待办项按回车时，自动插入下一行未完成待办前缀，例如 `- [ ] 内容`、`- [x] 内容` -> 下一行 `- [ ] `。
- 保留缩进，支持 `  - 子项` 和 `  - [ ] 子任务`。
- 在空列表项或空待办项按回车时退出列表，移除当前行 Markdown 前缀。
- `MarkdownBodyTextView` 只在 `UITextViewDelegate.shouldChangeTextIn` 中调用纯规则；SwiftUI binding 仍是正文状态源。
- 补充 XCTest 覆盖列表续写、待办续写、完成待办转未完成、空项退出、缩进保留、代码块内不接管、非折叠选区不接管和 UTF-16/emoji 边界。
- 同步 README、测试规范、核心流程、流程图和更新日志。
- GitHub Actions 结果包版本更新为 `v0.18`。

## 非目标

- 不新增 native macOS target，仍使用现有 Mac Catalyst。
- 不改 `JournalStore` 保存、排序、JSON 持久化格式或节流策略。
- 不改变 Markdown 预览解析规则和 `###` 小节识别。
- 不引入第三方 Markdown 编辑器或外部依赖。
- 不实现 Tab/Shift-Tab 缩进、智能引号/破折号配置或更多快捷键；这些留作后续独立轮次。
- 不把本机 GUI 启动、本机构建或本机 XCTest 作为本轮验收依据。

## 架构依据

- `MarkdownBodyTextView` 是正文输入桥接层，适合拦截 `UITextViewDelegate.shouldChangeTextIn`，但不应承载 Markdown 字符串规则。
- `MarkdownLineContinuation` 属于规则层，和 `MarkdownSnippetInsertion` 一样只消费字符串与 `NSRange`，输出新正文和新选区。
- `EntryEditorView`、`JournalStore`、`MarkdownPreviewView` 和统计模块不应知道回车续写细节。
- 云端仍由 `.github/workflows/ci-results.yml` 负责 static checks、generic iOS build、Mac Catalyst build、XCTest 和 artifact。

## 实现步骤

1. 新增 `MarkdownLineContinuation`，定义 `Result`，输出 `body` 和 `selectedRange`。
2. 实现 `apply(to:selectedRange:replacementText:) -> Result?`：
   - 只处理 `replacementText == "\n"`。
   - 非折叠选区返回 `nil`，交给系统默认行为。
   - 当前行不是 `- `、`* `、`+ `、`- [ ] `、`- [x] `、`- [X] ` 形态时返回 `nil`。
   - 位于 fenced code block 内时返回 `nil`。
   - 非空列表项插入 `\n` 加同缩进续写前缀。
   - 空列表项移除当前行前缀，退出列表。
3. 在 `MarkdownBodyTextView.Coordinator` 中实现 `textView(_:shouldChangeTextIn:replacementText:)`：
   - 若存在 marked text，返回 `true`。
   - 若纯规则返回 `Result`，手动更新 `textView.text`、`textView.selectedRange`、`text` binding 和 `selectedRange` binding，并返回 `false`。
   - 其他输入维持原有系统行为。
4. 新增 `MarkdownLineContinuationTests` 覆盖核心边界。
5. 更新 Xcode project，把新增 app source 和 test source 加入对应 target。
6. 更新 CI `VERSION` 为 `v0.18`。
7. 同步 README、测试规范、核心流程、流程图和更新日志。

## 验证要求

本轮人工要求以云端回传测试为准，不在本机执行 build、run 或 XCTest。Agent B 可做只读检查和 diff 审核，但不得把本机测试结果作为验收结论。

必须通过 GitHub Actions artifact 复判：

- workflow：`MD Journal CI Results`。
- artifact 名称必须使用 `mdjournal-ci-v0.18-main-<short_sha>-run<run_id>-attempt<run_attempt>`。
- manifest 必须记录 `version=v0.18`、`branch=main`、本轮 `commitSha`、`runId`、`runAttempt`、`staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome` 和 `testOutcome`。
- `junit.xml` 中失败数为 0，且 XCTest 没有被跳过。
- `xcodebuild.log` 和 `maccatalyst-build.log` 必须以 `** BUILD SUCCEEDED **` 结束。
- `xctest.log` 必须以 `** TEST SUCCEEDED **` 结束，并包含新增 `MarkdownLineContinuationTests` 通过记录。
- `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult`、`MDJournalTests.xcresult` 必须存在且 `Info.plist` 可解析。

Agent C 必须下载最新 `origin/main` 对应 run 的未加密 artifact 到 `/private/tmp/mdjournal-c-review-<run_id>/`，并核对：

- manifest 的 `version`、`branch`、`commitSha`、`runId`、`runAttempt` 和当前最新 `origin/main` 完全一致。
- `junit.xml` 失败数为 0，且 XCTest 没有被跳过。
- `static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log` 与 manifest outcome 一致。
- 新增 `MarkdownLineContinuationTests` 出现在 `xctest.log` 且全部通过。
- `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult` 和 `MDJournalTests.xcresult` 存在。
- 文档已记录 v0.18 的写作规则、验证结果、artifact 名称和遗留事项。

## 风险和禁止项

- 禁止绕过 `UITextView` binding，让 UIKit 成为第二套正文状态源。
- 禁止在 marked text / IME 组合输入期间强行改写正文。
- 禁止改变正文保存时序、`JournalStore` 集合排序或 JSON 数据格式。
- 禁止破坏 `MarkdownSnippetInsertion` 的光标/选区插入规则。
- 禁止用旧 run 或旧 artifact 冒充本轮 v0.18 云端验收。
