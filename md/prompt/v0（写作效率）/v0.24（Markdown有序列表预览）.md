# v0.24（Markdown 有序列表预览）

## 版本号

`v0.24`

## 版本分配依据

当前最新正式版本为 `v0.23`。本轮继续沿 `v0.x` 小步优化，承接 v0.23 的有序列表回车续写能力，把 Mac Catalyst 宽屏编辑/预览链路补齐到“写得顺、看得清”。本轮不改变数据模型、持久化、保存链路或统计口径。

## 背景

v0.23 已支持正文中 `数字. ` 有序列表的回车续写，但 `MarkdownBlockParser` 当前只把无序列表和待办解析为列表块，`MarkdownPreviewView` 也没有有序列表渲染分支。用户在 Mac Catalyst 宽屏下常会左侧编辑、右侧预览，有序列表输入后却在预览里退化为普通段落，写作体验不完整。

## 目标

- 扩展 `MarkdownBlockParser`，支持识别 `数字. ` 有序列表行。
- 解析结果保留每一项的原始编号文本和正文文本，用于预览按用户输入编号展示。
- `MarkdownPreviewView` 增加有序列表渲染分支，使用对齐的编号列和正文列展示。
- 保持无序列表、待办、引用、代码块、分割线、标题、段落和 `###` 小节分组行为不变。
- fenced code block 内的 `1. code` 不得被识别为有序列表。
- 新增或扩展 `MarkdownBlockParserTests`，覆盖有序列表解析、混合块 flush、代码块内不解析、`###` 小节分组中保留有序列表。
- 将 CI artifact 版本更新为 `v0.24`，并同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不支持 `1) ` 变体。
- 不做整段自动重编号，不改 v0.23 的回车续写规则。
- 不新增有序列表工具栏按钮、菜单项或快捷键。
- 不支持嵌套有序列表层级渲染；本轮只识别 leading whitespace trim 后的单层 `数字. `。
- 不引入第三方 Markdown renderer，不承诺完整 CommonMark。
- 不改变 `JournalStore` 保存、排序、节流或 JSON 持久化。
- 不新增 native macOS target。

## 架构依据

- `MarkdownBlockParser` 是轻量 Markdown 块级解析边界，新增语法必须在这里集中处理。
- `MarkdownPreviewView` 只消费解析结果并渲染，不负责保存数据或修改正文。
- `EntryEditorView`、`MarkdownBodyTextView` 和 `MarkdownLineContinuation` 已完成输入侧能力，本轮不应回改输入规则。
- 解析语义必须用 `MDJournalTests` 覆盖，最终通过 GitHub Actions artifact 复判。

## 实现步骤

1. 在 `MarkdownBlockParser.swift` 中新增有序列表块模型：
   - `MarkdownBlock` 增加 `.orderedList([OrderedListItem])`。
   - 新增 `OrderedListItem`，至少保存 `number` 和 `text`。
2. 扩展解析状态：
   - 新增 `orderedItems` 缓冲和 `flushOrderedList()`。
   - `flushInlineBlocks()` 同时 flush 段落、无序列表、待办和有序列表。
   - 有序列表行出现时 flush 段落、无序列表和待办，再追加有序项。
   - 普通段落、标题、引用、分割线、代码块和空行出现时正确 flush 有序列表。
3. 新增有序列表识别函数：
   - 只识别 trim leading whitespace 后的 `数字. `。
   - 编号文本保留为字符串，避免超大编号溢出，也保留用户输入的显示形式。
   - `1.` 没有空格时不识别。
4. 在 `MarkdownPreviewView.blockView(_:)` 中新增 `.orderedList` 分支：
   - 使用垂直列表逐项渲染。
   - 左侧编号列使用等宽/对齐的编号文本，右侧正文使用 `inlineMarkdown`。
   - 保持安静、工具型、8px 圆角以内的现有风格。
5. 扩展 `MarkdownBlockParserTests`：
   - `parse` 识别有序列表并和其他块顺序正确。
   - 有序列表与无序列表、待办、段落之间能正确 flush。
   - fenced code block 内 `1. code` 保持代码文本。
   - `parseDocument` 的 `sectionGroups` 能保留有序列表块。
   - `1.` 和 `1) ` 不识别为有序列表。
6. 更新 workflow `VERSION: v0.24`。
7. 同步 README、测试规范、核心流程、流程图和更新日志。
