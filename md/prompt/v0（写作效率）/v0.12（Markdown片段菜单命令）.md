# v0.12（Markdown 片段菜单命令）

## 版本分配依据

当前最新正式版本为 `v0.11`，已完成 Mac Catalyst 统计独立窗口。人工总目标要求继续优化写作体验、Mac 版桌面操作和高效写入；本轮沿 `v0.x` 小版本递增为 `v0.12`，聚焦 Markdown 片段从菜单和硬件键盘插入。

## 背景

现有编辑器已有 `MarkdownToolbar` 和 `MarkdownSnippet`，但片段插入主要依赖编辑区内工具栏，桌面菜单和硬件键盘路径不足。当前 `TextEditor(text:)` 没有项目内选区桥接，本轮不做光标处插入或包裹选中文本，避免扩大到 UIKit/AppKit 包装。

## 目标

- 新增“插入 Markdown”菜单，列出当前全部 `MarkdownSnippet`。
- 为每个片段提供唯一的低冲突键盘快捷键。
- 菜单命令通过 focused scene value 路由到当前 `EntryEditorView`，复用现有 `insertSnippet(_:)` 写入逻辑。
- 窄屏处于预览模式时，触发片段插入后自动切回编辑模式并聚焦正文。
- 工具栏按钮仍保持原有行为。
- 保持 snippet markdown 文本不变，不改变 JSON schema。
- GitHub Actions artifact 版本更新为 `v0.12`。

## 非目标

- 不实现光标位置插入。
- 不实现选中文本包裹。
- 不引入 UIKit/AppKit 文本编辑器桥接。
- 不改变 Markdown 预览解析语义或 `###` 小节规则。
- 不新增第三方依赖。

## 架构依据

- `MarkdownSnippet` 是片段标题、图标和 markdown contract 的集中边界。
- `MarkdownSnippetCommandShortcut` 是菜单层的 SwiftUI 快捷键映射，避免把 `EventModifiers` 等视图命令类型下沉到 snippet 模型。
- `EntryEditorView` 是正文编辑和片段插入的唯一 UI 边界。
- `MDJournalApp` 的 commands 负责桌面菜单和快捷键入口。
- focused scene value 只暴露当前编辑器插入动作，不绕过 `JournalStore` 和 `Binding<JournalEntry>` 状态流。

## 实现步骤

1. 保持 `MarkdownSnippet.markdown` 输出不变，复用 `MarkdownSnippet.title` 生成菜单标题。
2. 在 `MDJournalApp` 中新增 focused value：`insertMarkdownSnippetAction`。
3. 新增菜单层 `MarkdownSnippetCommandShortcut`，为每个片段提供唯一 `⌘⌥` 快捷键并加入 app target。
4. 新增“插入 Markdown”命令菜单，遍历 `MarkdownSnippet.allCases`，触发当前 focused 插入动作。
5. 在 `EntryEditorView` 通过 `.focusedSceneValue` 暴露 `insertSnippet(_:)`。
6. 修改 `insertSnippet(_:)`，插入前把 `mode` 切回 `.edit`，插入后继续聚焦正文。
7. 补充 `MarkdownSnippetTests`，覆盖片段顺序、快捷键唯一性和 markdown contract 不变。
8. 更新 CI `VERSION` 为 `v0.12`。
9. 同步 README、核心流程、流程图、测试规范和更新日志。
