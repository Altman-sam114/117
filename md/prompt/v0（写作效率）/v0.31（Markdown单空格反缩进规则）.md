# v0.31 / Markdown 单空格反缩进规则

## 版本分配依据

- 当前最新正式版本为 `v0.30`。
- 本轮继续围绕 Agent X 总目标中的“Mac 版写作效率、优美写入和高效性能”做小步优化。
- 目标是修正既有 Markdown 行缩进规则的边界行为，沿 `v0.x` 小版本递增为 `v0.31`。

## 背景

`MarkdownLineIndentation` 已集中处理正文编辑中的 Tab / Shift-Tab 行缩进。项目文档和 `v0.20` 记录都说明反缩进应删除一个 tab 或最多两个行首空格，但当前实现只删除一个 tab 或完整的两个空格。若一行只有一个行首空格，Mac 硬件键盘 Shift-Tab 不会产生结果，和既有规则描述不一致。

## 目标

- 让 `MarkdownLineIndentation` 的 `.outdent` 对行首一个空格也能删除该空格。
- 保持两个行首空格仍一次删除两个空格，行首 tab 仍一次删除一个 tab。
- 多行选区中可反缩进的行各自按规则处理；没有行可反缩进时仍返回 `nil`。
- 扩展 `MarkdownLineIndentationTests` 覆盖单空格和多行混合边界。
- 将 `.github/workflows/ci-results.yml` 的 `VERSION` 更新为 `v0.31`。
- 同步 README、测试规范、核心流程、流程图、本日志和本提示词。

## 非目标

- 不新增菜单、按钮或快捷键。
- 不修改 `MarkdownBodyTextView` 的 UIKit bridge、焦点、选区同步或 IME 逻辑。
- 不修改 Markdown 片段插入、回车续写、预览解析或统计逻辑。
- 不改变 JSON 持久化格式、`JournalStore` 保存链路或 deployment target。
- 不引入第三方依赖，不新增 native macOS target。

## 架构依据

- `MarkdownLineIndentation` 是纯规则层，负责根据正文、UTF-16 选区和方向生成替换结果。
- `MarkdownBodyTextView` 只负责捕获 Tab / Shift-Tab 并调用规则层，不应承载缩进策略。
- XCTest 可以直接覆盖该纯规则，不需要本地 UI 或模拟器交互。

## 实现步骤

1. Agent B 开始前同步最新 `origin/main`，确认当前分支为 `main` 且工作区无无关改动。
2. 在 `MarkdownLineIndentation.operation(for:direction:in:)` 的 `.outdent` 分支中保留现有优先级：
   - 若行首有两个空格，删除两个空格。
   - 否则若行首是 tab，删除一个 tab。
   - 否则若行首是一个空格，删除一个空格。
   - 其他情况返回 `nil`。
3. 在 `MarkdownLineIndentationTests` 新增单空格反缩进测试。
4. 新增多行混合测试，覆盖一个空格、两个空格、tab 和无缩进行。
5. workflow `VERSION` 改为 `v0.31`。
6. 文档更新只描述本轮真实行为，不扩大到 UI 入口或新快捷键。
