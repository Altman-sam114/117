# v0.19（Markdown 安全输入配置）

## 版本号

v0.19

## 版本分配依据

当前最新正式版本为 `v0.18`，已支持 Markdown 列表和待办回车续写。人工总目标继续要求推进“Mac 版、优美写入、高效性能”，且明确要求验证以 GitHub Actions 回传结果包为准，不走本机运行测试。本轮沿 `v0.x` 小版本递增为 `v0.19`，聚焦 Markdown 正文输入的安全性：避免智能引号、智能破折号和智能插入删除改写 Markdown 标记。

## 背景

`MarkdownBodyTextView` 使用 `UITextView` 承载正文输入。当前创建 `UITextView` 时未显式配置 smart dashes、smart quotes 和 smart insert/delete。Mac Catalyst 或硬件键盘输入 Markdown 时，系统输入 traits 可能把 `---`、反引号、普通引号或列表标记附近的空格自动替换，影响 Markdown 的可预测性。

## 目标

- 在 `MarkdownBodyTextView` 中新增集中配置方法，设置正文输入 traits：
  - `smartDashesType = .no`
  - `smartQuotesType = .no`
  - `smartInsertDeleteType = .no`
- `makeUIView` 创建正文 `UITextView` 时应用该配置；`updateUIView` 可重复应用，确保 traits 被外部重置后恢复。
- 新增 XCTest 覆盖该配置方法，确认 Markdown 安全输入 traits 均为 `.no`。
- 同步 README、测试规范、核心流程、流程图和更新日志。
- GitHub Actions 结果包版本更新为 `v0.19`。

## 非目标

- 不改正文内容、选区同步、回车续写、片段插入或 Markdown 预览规则。
- 不关闭中文输入、拼写检查、自动大写或 autocorrection；避免扩大输入体验影响。
- 不新增 native macOS target，仍使用现有 Mac Catalyst。
- 不改 `JournalStore` 保存、排序、JSON 持久化格式或节流策略。
- 不把本机 GUI 启动、本机构建或本机 XCTest 作为本轮验收依据。

## 架构依据

- `MarkdownBodyTextView` 是正文输入桥接层，输入 traits 属于该桥接层职责。
- 输入 trait 配置不应进入 `JournalStore`、`EntryEditorView`、`MarkdownPreviewView` 或 Markdown 规则层。
- 云端仍由 `.github/workflows/ci-results.yml` 负责 static checks、generic iOS build、Mac Catalyst build、XCTest 和 artifact。

## 实现步骤

1. 在 `MarkdownBodyTextView` 中新增 internal static 方法，例如 `configureMarkdownInputTraits(_:)`。
2. 在 `makeUIView` 设置字体、inset 等现有属性时调用该方法。
3. 在 `updateUIView` 中重复调用该方法，避免输入 traits 被重置。
4. 新增 `MarkdownTextInputConfigurationTests`，用 `UITextView` 调用配置方法并断言三个 smart traits 均为 `.no`。
5. 更新 Xcode project，把新增 test source 加入 `MDJournalTests` target。
6. 更新 CI `VERSION` 为 `v0.19`。
7. 同步 README、测试规范、核心流程、流程图和更新日志。

## 验证要求

本轮人工要求以云端回传测试为准，不在本机执行 build、run 或 XCTest。Agent B 可做只读检查和 diff 审核，但不得把本机测试结果作为验收结论。

必须通过 GitHub Actions artifact 复判：

- workflow：`MD Journal CI Results`。
- artifact 名称必须使用 `mdjournal-ci-v0.19-main-<short_sha>-run<run_id>-attempt<run_attempt>`。
- manifest 必须记录 `version=v0.19`、`branch=main`、本轮 `commitSha`、`runId`、`runAttempt`、`staticChecksOutcome`、`buildOutcome`、`macCatalystBuildOutcome` 和 `testOutcome`。
- `junit.xml` 中失败数为 0，且 XCTest 没有被跳过。
- `xcodebuild.log` 和 `maccatalyst-build.log` 必须以 `** BUILD SUCCEEDED **` 结束。
- `xctest.log` 必须以 `** TEST SUCCEEDED **` 结束，并包含新增 `MarkdownTextInputConfigurationTests` 通过记录。
- `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult`、`MDJournalTests.xcresult` 必须存在且 `Info.plist` 可解析。

- manifest 的 `version`、`branch`、`commitSha`、`runId`、`runAttempt` 和当前最新 `origin/main` 完全一致。
- `junit.xml` 失败数为 0，且 XCTest 没有被跳过。
- `static-checks.log`、`xcodebuild.log`、`maccatalyst-build.log`、`xctest.log` 与 manifest outcome 一致。
- 新增 `MarkdownTextInputConfigurationTests` 出现在 `xctest.log` 且全部通过。
- `MDJournal.xcresult`、`MDJournalMacCatalyst.xcresult` 和 `MDJournalTests.xcresult` 存在。
- 文档已记录 v0.19 的输入配置、验证结果、artifact 名称和遗留事项。
