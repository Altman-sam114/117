# v0.15（光标/选区插入 Markdown 片段）

## 版本号

v0.15

## 版本分配依据

当前最新正式版本为 `v0.14`，已完成 Mac Catalyst 写作菜单、写作工具栏和宽屏预览栏显示/隐藏。人工总目标要求继续推进“Mac 版、优美写入、高效性能”。v0.14 的明确遗留事项是 Markdown 片段仍只能追加到正文末尾，不支持光标位置插入或选中文本包裹。本轮沿 `v0.x` 小版本递增为 `v0.15`，聚焦写作效率的核心缺口。

## 背景

当前 `EntryEditorView.insertSnippet(_:)` 会先聚焦正文，再把片段追加到 `entry.body` 末尾。这个行为稳定，但在 Mac Catalyst 和硬件键盘写作场景中不自然：用户通常期望工具栏、菜单或快捷键在当前光标处插入 Markdown，选中文本时能包裹或逐行转换选区。

SwiftUI `TextEditor` 在当前 iOS 16 / Mac Catalyst 13 基线下不暴露可靠的选区绑定。本轮需要用最小 `UITextView` bridge 获取和同步正文光标/选区，同时把 Markdown 插入规则放进纯函数中测试，避免把字符串规则散落在视图层。

## 目标

- 新增可测试的 `MarkdownSnippetInsertion`，基于正文、片段和 `NSRange` 生成新正文与新选区。
- Markdown 片段从“固定追加到末尾”改为“按当前光标或选区插入”。
- 加粗、斜体、代码块支持包裹选中文本。
- 引用、列表、待办支持对多行选区逐行加前缀。
- 空选区保持现有片段语义，并把光标或选区放在自然继续编辑的位置。
- 用最小 `UIViewRepresentable` 包装 `UITextView`，只负责正文绑定、选区同步和焦点同步。
- 保持菜单、Mac Catalyst 写作工具栏、键盘快捷键和正文工具栏都复用同一 `insertSnippet(_:)`。
- 保持 `JournalStore`、`JournalEntry` JSON schema、Markdown 预览解析和统计口径不变。
- GitHub Actions 结果包版本更新为 `v0.15`。

## 非目标

- 不新增 native macOS target，Mac 路径仍是 Mac Catalyst。
- 不引入第三方 Markdown 编辑器或外部依赖。
- 不实现完整 Markdown AST 编辑器。
- 不改变 `MarkdownSnippet.markdown` 原始片段字符串契约，除非测试同步明确说明。
- 不绕过 `JournalStore` 批量写日记集合。
- 不新增云同步、账号、服务器或持久化索引。

## 架构依据

- `EntryEditorView` 是正文编辑、片段插入和编辑/预览布局边界。
- `MarkdownSnippet` 是片段类型和默认片段文本契约。
- `MarkdownToolbar`、Mac Catalyst 写作工具栏和 `MDJournalApp` 的“插入 Markdown”菜单已经统一调用 `EntryEditorView.insertSnippet(_:)`。
- 新增 `MarkdownSnippetInsertion` 作为规则层，不接触视图、持久化或解析器。
- 新增 `MarkdownBodyTextView` 作为最小 `UITextView` bridge，只替换正文输入控件，不承担 Markdown 规则。

## 实现步骤

1. 新增 `MarkdownSnippetInsertion`，定义 `Result` 结构，输出 `body` 和 `selectedRange`。
2. 让 `MarkdownSnippetInsertion.apply(snippet:to:selectedRange:)` 使用 `Range(nsRange, in:)` 转换 UTF-16 `NSRange`，非法 range 时安全 clamp 到正文末尾。
3. 为普通空选区插入保留现有片段字符串，并返回适合继续编辑的新光标位置。
4. 为选区包裹类片段实现包裹逻辑：加粗、斜体和代码块。
5. 为多行转换类片段实现逐行前缀逻辑：引用、列表、待办。
6. 新增 `MarkdownBodyTextView`，用 `UITextView` 同步 `text`、`selectedRange` 和 `isFocused`，只在值确实不一致时写回 UIKit，避免光标跳动。
7. 在 `EntryEditorView` 中新增正文选区状态，把 `TextEditor` 替换为 `MarkdownBodyTextView`。
8. 更新 `insertSnippet(_:)`，调用 `MarkdownSnippetInsertion` 后写回 `entry.body` 和选区。
9. 扩展 `MarkdownSnippetTests`，覆盖光标中间插入、选区包裹、多行前缀、非法 range、中文/emoji UTF-16 range 和现有 raw markdown contract。
10. 更新 Xcode project，把新增 Swift 文件加入 app sources；若新增测试文件，也加入 test sources。
11. 更新 CI `VERSION` 为 `v0.15`。
12. 同步 README、测试规范、核心流程、流程图和更新日志。

## 风险和禁止项

- `UITextView` bridge 不能成为第二套状态源；SwiftUI binding 仍是正文源头。
- `updateUIView` 不能无条件重设 `textView.text`，避免输入时光标跳动。
- `NSRange` 必须按 UTF-16 安全转换，遇到非法 range 时 clamp。
- 禁止改变 `JournalEntry` JSON schema、`JournalStore` 保存链路、Markdown 预览解析语义或统计口径。
- 禁止把本轮描述成完整 Markdown 编辑器。
- 禁止用旧 run 或旧 artifact 冒充本轮 v0.15 云端验收。
