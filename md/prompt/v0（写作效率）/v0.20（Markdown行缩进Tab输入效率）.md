# v0.20（Markdown 行缩进 Tab 输入效率）

## 版本号

`v0.20`

## 版本分配依据

当前最新正式版本为 `v0.19`。本轮继续沿 `v0.x` 写作效率阶段推进，在已完成 Markdown 列表回车续写和安全输入配置之后，补齐 Mac Catalyst / 硬件键盘常用的 Tab 与 Shift-Tab 行缩进能力。

## 背景

MD Journal 已经通过 `MarkdownBodyTextView` 将正文输入切换到 `UITextView` bridge，并用纯规则承载 Markdown 片段插入、列表回车续写和输入 traits 配置。Mac Catalyst 下用户会自然期待在正文编辑时用 Tab 缩进当前行或多行选区，用 Shift-Tab 反缩进。仅依赖 `UITextViewDelegate.shouldChangeTextIn` 不能可靠区分 Shift-Tab，因此需要小范围扩展正文输入控件，同时保持字符串变换规则可单元测试。

## 目标

- 新增纯规则 `MarkdownLineIndentation`，负责 Markdown 正文行级缩进和反缩进。
- 在 `MarkdownBodyTextView` 中接入 Tab / Shift-Tab：普通 Tab 缩进当前行或选中多行，Shift-Tab 反缩进。
- 缩进单位使用两个空格，和现有列表续写缩进示例保持一致。
- 支持 UTF-16 `NSRange` 光标/选区，避免 emoji 等字符导致选区漂移。
- 新增 `MarkdownLineIndentationTests` 覆盖单行、多行、选区结束边界、emoji、空反缩进和混合缩进。
- 将 CI artifact 版本更新为 `v0.20`，并同步 README、测试规范、核心流程、流程图和更新日志。

## 非目标

- 不新增 native macOS target，Mac 版本仍走 Mac Catalyst。
- 不改变 `JournalEntry`、本地 JSON schema 或 `JournalStore` 保存链路。
- 不扩展 Markdown 预览语法。
- 不改变回车续写、Markdown 片段插入或输入 traits 的既有语义。
- 不引入第三方依赖。

## 架构依据

- `MarkdownBodyTextView` 负责 UIKit bridge、焦点、选区和输入入口。
- `MarkdownLineIndentation` 必须是纯字符串规则，不访问 SwiftUI、UIKit、JSON 或 `JournalStore`。
- `JournalStore` 仍是唯一日记集合修改和本地持久化入口，本轮只通过正文 binding 写回现有编辑链路。

## 实现步骤

1. 新增 `MarkdownLineIndentation`：
   - `Direction` 包含 `.indent` 和 `.outdent`。
   - `Result` 包含更新后的正文和 `NSRange`。
   - `.indent` 对当前行或选中多行插入两个空格。
   - `.outdent` 对当前行或选中多行删除最多一个缩进单位：两个空格或一个 tab。
   - 选区若刚好结束在下一行行首，不额外处理下一行。
2. 更新 `MarkdownBodyTextView`：
   - 使用轻量 `UITextView` 子类暴露 Tab 和 Shift-Tab `UIKeyCommand`。
   - `shouldChangeTextIn` 继续处理普通 Tab fallback。
   - IME marked text 存在时不拦截。
   - 成功应用规则后同步 `textView.text`、`textView.selectedRange`、SwiftUI text binding 和 selectedRange binding。
3. 新增 `MarkdownLineIndentationTests`：
   - 单光标 Tab 缩进当前行。
   - 多行选区 Tab 逐行缩进。
   - 选区结束在下一行行首时不缩进后续行。
   - emoji / UTF-16 光标保持。
   - Shift-Tab 删除两个空格或一个 tab。
   - 混合缩进多行反缩进。
   - 无可反缩进行返回 nil。
4. 更新 Xcode project，把新 utility 加入 app target，把新 test 加入 `MDJournalTests` target。
5. 更新 workflow `VERSION: v0.20`。
6. 同步 README、测试规范、核心流程、流程图和更新日志。

## 本地轻量检查要求

人工已明确要求本轮不要在本机运行构建、测试或 app。因此 Agent B 不执行本机 `xcodebuild build`、`xcodebuild test`、模拟器或 app 启动；最终验收只以 GitHub Actions 回传 artifact 为准。

如需记录本地动作，只记录只读检查、git 状态和必要的提交/推送命令；不得用本地输出冒充云端验收。

## main 直推要求

- 开始前确认当前分支是 `main`。
- 执行 `git fetch origin` 和 `git pull --ff-only origin main`，确保基于最新 `origin/main`。
- 只提交本轮相关文件。

- workflow 名称：`MD Journal CI Results`。
- artifact 名称应包含 `v0.20`、`main`、短 SHA、run id 和 run attempt。

## 风险和禁止项

- 禁止把 Shift-Tab 仅描述为 delegate replacement text 行为；必须通过 key command 或等价键盘入口承载。
- 禁止在 IME marked text 存在时改写正文。
- 禁止破坏 `MarkdownLineContinuation` 回车续写、`MarkdownSnippetInsertion` 选区插入、`###` 小节识别和本地 JSON 保存。
- 禁止新增第三方依赖或 native macOS target。
- 禁止伪造本机测试、云端 run 或 artifact 验收结果。
